P1 — Frequently Asked Questions

You MUST know these for any React + TypeScript role.

1
What are the advantages of TypeScript over JavaScript for React projects?IntermediateP1
Basic Answer

TypeScript adds: catching bugs at compile time, better IDE support (autocomplete, refactoring), self-documenting code through types, safer refactoring, and catching null/undefined errors. For React: typed props prevent passing wrong data, typed state prevents invalid state transitions, and typed events prevent handler errors.

2
Explain the difference between interface and type alias. When to use each in React?IntermediateP1
Basic Answer

Both define shapes. Key differences: interfaces can be extended (declaration merging), types can do unions/intersections (more flexible). React convention: use interface for public API (props, state) and type for unions, complex types, and utility types. But consistency matters more.

interface ButtonProps { variant: 'primary' | 'secondary'; children: ReactNode; onClick?(): void; } type Status = 'idle' | 'loading' | 'success' | 'error'; type ApiResponse = { data: T; status: Status };
3
What are generics? Implement a generic React component.AdvancedP1
Basic Answer

Generics parameterize types, allowing components to work with different types safely. Essential for reusable React components like lists, tables, and selects.

interface ListProps { items: T[]; renderItem: (item: T) => ReactNode; keyExtractor: (item: T) => string; } function List({ items, renderItem, keyExtractor }: ListProps) { return ( <ul> {items.map(item => ( <li key={keyExtractor(item)}>{renderItem(item)}</li> ))} </ul> ); }
4
What are union types and intersection types? Give React examples.IntermediateP1
Basic Answer

Union (|): value can be one of several types. Intersection (&): value must satisfy all types. In React: unions for component variants, intersection for extending props.

type ButtonVariant = 'primary' | 'secondary' | 'ghost'; type WithChildren = T & { children: ReactNode };
5
Explain utility types: Partial, Required, Pick, Omit, Record, Readonly.AdvancedP1
Basic Answer

Partial — all optional. Required — all required. Pick — select keys. Omit — exclude keys. Record — key-value mapping. Readonly — all readonly. In React: Partial for form updates, Pick/Omit for prop subsets, Record for lookup maps.

6
What is the difference between any and unknown?IntermediateP1
Basic Answer

any opts out of type checking entirely. unknown is type-safe — you must narrow it before use (typeof, type guard, assertion). Always prefer unknown over any in React code. Use unknown for API responses, form data, and external inputs that need runtime validation.

7
How does TypeScript handle type inference? When should you use explicit typing in React?IntermediateP1
Basic Answer

TypeScript infers types from values. Always type function parameters and return types explicitly. Use inference for local variables, useState initial values, and simple data structures. In React, always type props, state, and event handlers explicitly.

8
What are mapped types? Create a custom mapped type for React.ExpertP1
Basic Answer

Mapped types create new types by transforming properties of an existing type. Useful for creating variant props, state types, and theme types in React.

type Theme = { primary: string; secondary: string; background: string }; type ThemeToken = `--color-${keyof Theme & string}`; type ThemeVars = Record;
9
What are conditional types? Explain with infer keyword.ExpertP1
Basic Answer

Conditional types select types based on conditions: T extends U ? X : Y. The infer keyword allows extracting types from complex structures. Used in React types like ComponentProps, ReturnType, and extracting prop types.

10
How do you type API responses in React? Design a generic API response type.AdvancedP1
Basic Answer

Use discriminated unions for API states and generics for type-safe data.

type ApiState = | { status: 'idle' } | { status: 'loading' } | { status: 'success'; data: T } | { status: 'error'; error: string }; function useApi(url: string): ApiState { // Implementation }
11
What are type guards? Create a custom type guard for React.AdvancedP1
Basic Answer

Type guards narrow types at runtime. Common guards: typeof, instanceof, in, discriminated unions. Custom guards use is return type. In React, useful for type-safe event handlers, form validation, and conditional rendering.

interface SuccessResponse { status: 'ok'; data: unknown; } interface ErrorResponse { status: 'error'; message: string; } type ApiResponse = SuccessResponse | ErrorResponse; function isSuccess(response: ApiResponse): response is SuccessResponse { return response.status === 'ok'; }
12
How do access modifiers (public, private, protected) work in TypeScript classes?IntermediateP1
Basic Answer

TypeScript access modifiers control visibility with class-based components (legacy). With hooks, classes are less common, but understanding them helps with legacy codebases and some library APIs. public — accessible anywhere (default). private — only within the class. protected — within class and subclasses.

13
What are abstract classes vs interfaces in React?AdvancedP1
Basic Answer

Abstract classes can have implementation. Interfaces are purely structural. In React, prefer interfaces for component contracts. Abstract classes are useful for class-based service layers or test utilities.

14
Explain keyof operator and how it works with generics in React.AdvancedP1
Basic Answer

keyof T yields a union of T's keys. Used in React for: type-safe form field names, sorting by keys, and generic table/component columns. keyof typeof pattern creates types from const objects.

15
What are discriminated unions? How do they enable type-safe state in React?AdvancedP1
Basic Answer

Discriminated unions use a literal property (discriminant) to differentiate between types. TypeScript narrows based on the discriminant. In React, they're the gold standard for modeling complex state: form states, multi-step wizards, API response states.

P2 — Commonly Asked Questions

16
What is enum? Compare numeric, string, and const enums.IntermediateP2
Basic Answer

Enums define named constants. Numeric enums auto-increment. String enums require explicit values. Const enums are compiled away (no runtime overhead). In React, union types ('a' | 'b') are often preferred over enums for props because they're more tree-shakable.

17
What are literal types? Give a React example.IntermediateP2
Basic Answer

Literal types are exact values: 'primary' | 'secondary', true, 42. Used in React for component props where only specific values are valid. Combined with as const for type-safe configurations.

18
What is the satisfies operator? When to use it?AdvancedP2
Basic Answer

satisfies checks a value matches a type WITHOUT widening. Keeps the most specific type for inference while validating against a broader type. In React: validating theme objects, config objects, or tailwind config extensions.

19
What are template literal types? Give a real React example.AdvancedP2
Basic Answer

Template literal types create string types using template syntax: type EventName = `on${Capitalize}`. In React: creating type-safe CSS class builders, event handler names, or Tailwind CSS class generation.

20
What are tuple types and variadic tuple types?AdvancedP2
Basic Answer

Tuples are fixed-length arrays with typed positions: [string, number]. Variadic tuples ([...T[], U]) handle dynamic lengths. In React: useState returns a tuple [T, Dispatch], useReducer returns [T, Dispatch].

21
How does TypeScript handle async/await return types?IntermediateP2
Basic Answer

An async function always returns Promise. The T is inferred from the return value. In React: useEffect callbacks can't be async directly (they return cleanup, not Promise). Define async function inside and call it.

22
What is the never type? When does it occur in React?AdvancedP2
Basic Answer

never represents values that never occur. Used in: exhaustive checks in switch statements (discriminated unions), unreachable code after TypeScript narrowing, and generic utility types. In React: exhaustive check on enum-like props, exhaustive reducer cases.

23
How do you type event handlers in React?IntermediateP2
Basic Answer

React provides synthetic event types: React.MouseEvent, React.ChangeEvent, React.FormEvent, React.KeyboardEvent. Use React.EventHandler<> for callback types.

24
How do you design type-safe Page Objects or component tests in React?AdvancedP2
Basic Answer

Use generics to create type-safe test utilities. Examples: type-safe render helpers, type-safe query functions, and type-safe test data factories.

25
Explain as const assertions and their use in React.AdvancedP2
Basic Answer

as const marks values as readonly literal types. In React: defining variant mappings, color palettes, action types, and configuration objects where exact values must be preserved for type inference.

26
What is module resolution? How does paths mapping work in tsconfig.json?IntermediateP2
Basic Answer

Module resolution determines how imports are found. paths in tsconfig.json creates aliases: "@components/*": ["src/components/*"]. In React projects, path aliases simplify imports: import Button from '@components/Button' instead of ../../components/Button.

27
How do you create a type-safe configuration system for React apps?AdvancedP2
Basic Answer

Use as const + satisfies + typeof to create type-safe configs. Example: environment variable validation, feature flags, theme configurations, and API endpoint definitions.

28
What is NonNullable and how does it prevent null reference errors?IntermediateP2
Basic Answer

NonNullable removes null and undefined from T. In React: useful for removing null from optional callbacks, cleaning up types from API responses, and ensuring safe access to element refs.

29
How do decorators work in TypeScript? React use case?ExpertP2
Basic Answer

Decorators are annotations for classes (experimental in TS). Used in legacy class-based React components for HOCs, MobX observables, and Angular-like patterns. Not commonly used in modern functional React.

30
What is Extract and Exclude utility types?AdvancedP2
Basic Answer

Extract — extracts types from T that are assignable to U. Exclude — removes types from T that are assignable to U. In React: filtering union types for conditional props, removing certain variants from component prop types.

31
How does typeof work in type context vs value context?AdvancedP2
Basic Answer

In value context: returns runtime type string. In type context (typeof x): returns the TypeScript type of x. Used to extract types from values, modules, and configuration objects. typeof import('./module') gives module type.

32
What is InstanceType and when would you use it?AdvancedP2
Basic Answer

InstanceType extracts the instance type from a constructor type. Useful for typing class instances and inferring component prop types from class components (legacy React).

33
What are Parameters and ReturnType utility types?AdvancedP2
Basic Answer

Parameters — extracts parameter types as a tuple. ReturnType — extracts the return type. In React: typing wrapper components that spread props, creating typed event handler generics, and extracting API function signatures.

P3 — Occasionally Asked (Valuable Context)

34
How do you create a deep Partial or deep Readonly type?ExpertP3
Basic Answer

Use recursive mapped types: type DeepPartial = T extends object ? { [P in keyof T]?: DeepPartial } : T. In React: form state with nested objects, deeply nested theme overrides.

35
What is brand/nominal typing in TypeScript? How do you implement it?ExpertP3
Basic Answer

Brand typing creates distinct types from the same primitive: type UserId = string & { __brand: 'UserId' }. Prevents passing wrong ID types accidentally. In React: type-safe IDs for different entities in state management.

36
How does declaration merging work with interfaces?AdvancedP3
Basic Answer

Multiple interface declarations with the same name merge into one. Types don't merge. In React: used to extend module typings (e.g., adding to Window, augmenting CSS properties). Also used by libraries like Styled Components for theme augmentation.

37
What are call signatures and construct signatures?ExpertP3
Basic Answer

Call signatures describe a function's params and return type. Construct signatures describe a constructor with new. Used when typing function overloads, factory patterns, and some React component patterns (like React.FC type).

38
How do you type dynamic object keys in React?AdvancedP3
Basic Answer

Use index signatures: { [key: string]: T } or Record. For specific keys from a union: type ValidKeys = 'name' | 'email'; type FormState = Record. In React: dynamic form state, lookup tables, error messages object.

39
How do you type children in React? Compare ReactNode vs ReactElement vs JSX.Element.IntermediateP3
Basic Answer

ReactNode — most permissive (includes strings, numbers, null, undefined, arrays, portals). ReactElement — only JSX elements with type and props. JSX.Element — a ReactElement with any props. For most components, use ReactNode for children.

40
How do you type React hooks with generics?AdvancedP3
Basic Answer

Custom hooks can use generics for type-safe return values. Example: useLocalStorage(key: string, initial: T): [T, (value: T) => void], useFetch(url: string): ApiState, useForm>(initial: T).
