P1 — Frequently Asked Questions

1
Compare Tailwind CSS vs CSS Modules vs CSS-in-JS. When to use each?IntermediateP1
Basic Answer

Tailwind CSS: utility-first, fast prototyping, small production CSS (purging), consistent design system, popular for new projects. CSS Modules: scoped by default, zero runtime, good for component libraries, familiar CSS syntax. CSS-in-JS (Styled Components/Emotion): dynamic styles via props, runtime overhead, less popular in 2024+. Tailwind is currently most popular for React projects.

2
Explain Flexbox layout — common patterns and properties.IntermediateP1
Basic Answer

Flexbox is one-dimensional layout. Container: display: flex, flex-direction (row/column), justify-content (main axis), align-items (cross axis), flex-wrap. Items: flex (grow/shrink/basis shortcut), align-self, order. Common patterns: navbar, card grid, centering (justify-content: center; align-items: center), sidebar + content.

3
Explain CSS Grid — when to use it over Flexbox.IntermediateP1
Basic Answer

CSS Grid is two-dimensional — rows AND columns simultaneously. Use Grid for: page layouts, complex card grids, dashboards, media galleries. Use Flexbox for: one-dimensional layouts, navigation bars, centering, small component layouts. Key Grid properties: grid-template-columns, gap, grid-area, minmax(), auto-fill/auto-fit.

4
How do you handle responsive design in React?IntermediateP1
Basic Answer

Approaches: CSS media queries (Tailwind responsive prefixes), useMediaQuery custom hook for React-responsive, Container queries (@container — new!), CSS Grid auto-fill/auto-fit, and clamp() for fluid typography. Mobile-first approach: start with base styles, add breakpoints as needed. Tailwind: sm:, md:, lg:, xl:, 2xl:.

5
How do you implement theming and dark mode in React?AdvancedP1
Basic Answer

Approaches: CSS custom properties (variables) + class toggle. Tailwind: dark: prefix with darkMode: 'class'. CSS-in-JS: ThemeProvider (Styled Components). Context API + CSS variables for runtime switching. Persist preference in localStorage. Use prefers-color-scheme media query for system preference detection.

6
What is BEM? Other CSS naming conventions?IntermediateP1
Basic Answer

BEM (Block-Element-Modifier): .block__element--modifier. Other approaches: SMACSS, OOCSS, Utility-first (Tailwind), CSS Modules (scoped by default), CSS-in-JS (locally scoped). In modern React, BEM is less common — CSS Modules and Tailwind have better scoping and DX.

7
How do CSS specificity conflicts occur? How do you prevent them in React?IntermediateP1
Basic Answer

Specificity: inline > ID > class/attribute/pseudo-class > element. In React, use CSS Modules or Tailwind to avoid specificity wars. Avoid !important. If using global CSS, use specific selectors. CSS Modules generate unique class names automatically. Tailwind avoids specificity by using only single-class selectors.

8
What are CSS custom properties (variables)? How do they enable theming?IntermediateP1
Basic Answer

CSS variables (--var-name: value) enable runtime theming without rebuild. Use var(--var-name, fallback). Set on :root or override in selectors. currentColor inherits the current text color. In React, update CSS variables via style.setProperty('--primary', color) or set style prop on root element.

9
What are CSS animations and transitions? How to use them in React?IntermediateP1
Basic Answer

Transitions: property change over time (transition: opacity 0.3s ease). Animations: keyframe-based (@keyframes fadeIn { from { opacity: 0 } to { opacity: 1 } }). In React: CSS transitions triggered by state changes (add/remove classes), Framer Motion for complex animations, AnimatePresence for mount/unmount. Performance: prefer transform and opacity for GPU-accelerated animations.

10
How do you build a design system in React?ExpertP1
Basic Answer

Start with: design tokens (colors, typography, spacing), component library (Button, Input, Select, Modal), documentation (Storybook), and testing. Use: Tailwind CSS for utility layer, Radix UI/Headless UI for accessible primitives, Storybook for documentation, Chromatic for visual regression. Key: consistent API, accessibility built-in, versioned releases.

P2 — Commonly Asked Questions

11
Explain the CSS box model and how it affects layout.BeginnerP2
Basic Answer

Content → Padding → Border → Margin. box-sizing: border-box includes padding and border in width/height (recommended — reset on all elements). Margin collapsing: vertical margins collapse. Use box-sizing: border-box globally via reset for predictable sizing.

12
What is position in CSS? Explain each value.BeginnerP2
Basic Answer

static (default) — normal flow. relative — offset relative to normal position. absolute — relative to nearest positioned ancestor. fixed — relative to viewport. sticky — toggles between relative and fixed based on scroll. z-index controls stacking — only works on positioned elements.

13
How does display property work? Compare block, inline, inline-block, none.BeginnerP2
Basic Answer

block — full width, new line, respects width/height. inline — content width, no new line, ignores width/height. inline-block — like inline but respects width/height. none — removed from layout (vs visibility: hidden which preserves space).

14
What are pseudo-classes and pseudo-elements? Give React examples.IntermediateP2
Basic Answer

Pseudo-classes (:hover, :focus, :active, :disabled, :nth-child, :not()) — element states. Pseudo-elements (::before, ::after, ::placeholder, ::selection) — create virtual elements. In React, pseudo-classes work naturally with CSS/Tailwind. Pseudo-elements are CSS-only — can't be targeted by React event handlers.

15
What is CSS containment? How does it improve performance?ExpertP2
Basic Answer

contain: layout style paint size isolates a subtree from the rest of the page. The browser can skip layout/style calculations outside the contained element when something inside changes. content-visibility: auto + contain-intrinsic-size for virtual scrolling without libraries. Huge performance boost for long pages/lists.

16
What is the :has() CSS selector? How does it change CSS?AdvancedP2
Basic Answer

:has() is the "parent selector" — selects an element if it contains certain children. card:has(img) styles cards with images differently. form:has(:invalid) highlights forms with errors. It enables state-based styling without JavaScript. Supported in all modern browsers.

17
What are Container Queries? How do they differ from Media Queries?AdvancedP2
Basic Answer

Container queries let components respond to their CONTAINER's size, not the viewport. @container (min-width: 400px) { .card { ... } }. Define container: container-type: inline-size. This is game-changing for reusable components — they can adapt based on where they're placed, not just viewport size.

18
What are the CSS Cascade Layers? @layerExpertP2
Basic Answer

@layer lets you explicitly control the cascade order: @layer base, components, utilities. Tailwind uses layers internally (@tailwind base/components/utilities). Helps prevent specificity wars — you can ensure framework styles don't override custom styles without !important.

19
How do you create responsive typography?IntermediateP2
Basic Answer

clamp(min, preferred, max): font-size: clamp(1rem, 2.5vw, 2rem). Viewport units (vw, vh). CSS locks (linear interpolation between breakpoints). Tailwind: text-base md:text-lg lg:text-xl. rem for accessibility (respects user font-size settings). Fluid typography is the modern approach.

20
Explain the calc() function and its uses in layouts.IntermediateP2
Basic Answer

calc() performs calculations in CSS: width: calc(100% - 200px), height: calc(100vh - 60px). Combines different units. Useful for: sidebar layouts, full-height minus header, fluid grids with fixed gutters.

21
What is will-change and when should you use it?AdvancedP2
Basic Answer

will-change: transform, opacity hints to the browser that an element will animate. Creates a compositing layer (GPU-accelerated). Use sparingly on elements that ARE about to animate — don't apply to everything. Overuse consumes GPU memory. Prefer transform and opacity animations which are already GPU-friendly.

22
How do you handle text overflow and ellipsis in CSS?BeginnerP2
Basic Answer

Single line: overflow: hidden; text-overflow: ellipsis; white-space: nowrap. Multi-line: display: -webkit-box; -webkit-line-clamp: 2; -webkit-box-orient: vertical; overflow: hidden. Tailwind: truncate for single, line-clamp-2 for multi-line.

23
How do you create a loading skeleton in CSS?IntermediateP2
Basic Answer

Use gradient animation: @keyframes shimmer { 0% { background-position: -200% } 100% { background-position: 200% } }. Apply to placeholder elements with background: linear-gradient(...). Libraries: react-loading-skeleton. Tailwind + custom animation for lightweight implementation.

24
What are modern CSS features you're excited about?AdvancedP2
Basic Answer

Container Queries, :has() selector, CSS Nesting (no more preprocessors needed), @scope (scoped styles), scroll-timeline (scroll-driven animations), view-transitions-api (SPA-like transitions), subgrid, accent-color, color-mix(), text-wrap: balance. These are making CSS more powerful than ever.

P3 — Occasionally Asked (Valuable Context)

25
Explain CSS Houdini and the Paint API.ExpertP3
Basic Answer

Houdini gives developers access to the CSS engine via JavaScript. Paint API: CSS.paintWorklet.addModule('myPaint.js') for custom paint (like background: paint(myGradient)). Typed OM: type-safe CSS values. Layout API: custom layout modes. Still experimental — limited browser support but powerful for polyfills.

26
How do you implement CSS parity for SSR (no Flash of Unstyled Content)?AdvancedP3
Basic Answer

CSS-in-JS: collect styles during SSR, inject into HTML (Styled Components ServerStyleSheet). CSS Modules: import CSS in components, Next.js handles SSR automatically. Tailwind: extract CSS at build time, works with SSR. FOUC prevention: include critical CSS in <head>.

27
What are the Core Web Vitals and how does CSS affect them?AdvancedP3
Basic Answer

LCP (Largest Contentful Paint): image optimization, font loading. FID/INP (Interaction to Next Paint): avoid long tasks, optimize animations. CLS (Cumulative Layout Shift): set width/height on images, aspect-ratio, avoid layout shifts from fonts. CSS affects all three — font loading, image sizing, animation performance.

28
How does content-visibility improve rendering performance?ExpertP3
Basic Answer

content-visibility: auto skips rendering of off-screen elements. The browser only paints elements in/near the viewport. Combine with contain-intrinsic-size to prevent scrollbar jumps. Can reduce initial render time by 30-50% for long pages. Use on: long lists, FAQ sections, comments, any content below the fold.

29
What is @container style queries?ExpertP3
Basic Answer

Container style queries check computed style values, not just sizes: @container style(--theme: dark) { ... }. Enables components to respond to CSS custom property values. Different from container size queries — this is about style-based theming at the container level.

30
How do you optimize CSS for production?AdvancedP3
Basic Answer

Minification, tree-shaking unused CSS (Tailwind's purge), code splitting CSS per route, critical CSS inlining (above-the-fold), CSS containment for scoped rendering, using @layer for cascade control, and removing unused CSS with PurgeCSS. PostCSS for optimization pipeline.
