🔴 P1 — Easy Problems

1
Two Sum — Find indices of two numbers that add to target.P1
Basic Answer

Use a Map to track seen numbers: const map = new Map(); for (let i = 0; i < nums.length; i++) { const complement = target - nums[i]; if (map.has(complement)) return [map.get(complement), i]; map.set(nums[i], i); }. O(n) time, O(n) space. Most commonly asked problem in frontend interviews.

2
Valid Parentheses — Check if brackets are correctly matched.P1
Basic Answer

Use a stack: push opening brackets, pop and match closing brackets. O(n) time, O(n) space. Must-know pattern for frontend interviews. Also tests understanding of stack data structure.

3
Valid Palindrome — Check if string is palindrome.P1
Basic Answer

Two-pointer approach: clean string (alphanumeric only, lowercase), compare left and right moving inwards. O(n) time, O(1) space with two pointers. Tests string manipulation and two-pointer pattern.

4
FizzBuzz — Classic screening question.P1
Basic Answer

Loop 1 to n, check divisibility by 3 and 5. Tests basic iteration and conditionals. Many candidates overcomplicate this — keep it simple.

5
Contains Duplicate — Check if array has duplicates.P1
Basic Answer

Use a Set: O(n) time, O(n) space. return new Set(nums).size !== nums.length. Tests understanding of Set and time complexity.

6
Valid Anagram — Check if strings are anagrams.P1
Basic Answer

Count character frequencies with a Map or 26-length array. O(n) time. Tests hash map pattern useful for data analysis in frontend.

7
Binary Search — Implement on sorted array.P1
Basic Answer

Left/right pointers, calculate mid, compare, narrow range. O(log n). Fundamental algorithm. Must-know for any software engineering role.

8
Reverse a Linked List.P1
Basic Answer

Iterative: three pointers (prev, current, next). O(n) time, O(1) space. Recursive also possible. Tests pointer manipulation and recursion.

9
Climbing Stairs — Count ways to climb n steps (1 or 2).P1
Basic Answer

Dynamic programming: dp[i] = dp[i-1] + dp[i-2], which is just Fibonacci. O(n) time, O(1) space with two variables. Tests DP fundamentals.

10
Maximum Subarray (Kadane's Algorithm).P1
Basic Answer

Track current sum and max sum: currentSum = Math.max(nums[i], currentSum + nums[i]); maxSum = Math.max(maxSum, currentSum). O(n) time, O(1) space. Tests greedy/DP thinking.

11
Move Zeroes — Move all zeros to end maintaining order.P1
Basic Answer

Two-pointer: one for non-zero placement, one for scanning. Swap non-zero elements forward. O(n) time, O(1) space. Tests two-pointer pattern.

12
Palindrome Number — Check integer palindrome without string conversion.P1
Basic Answer

Reverse half of the number: while (x > reversed) { reversed = reversed * 10 + x % 10; x = Math.floor(x / 10); }. Return x === reversed || x === Math.floor(reversed / 10). O(log n) time.

🟠 P2 — Medium Problems

13
Implement Debounce — Limit function call rate.P2
Basic Answer

Returns a function that delays execution until after delay ms of inactivity. Clear previous timer on each call. Tests closures, setTimeout, and this/args handling.

14
Implement Throttle — Ensure max one call per interval.P2
Basic Answer

Track last execution time. Only call if enough time has passed. Uses closures and timestamps. Tests understanding of timing and function wrapping.

15
Implement Promise.all — Wait for all promises.P2
Basic Answer

Return a Promise that resolves when all inputs resolve, or rejects on first rejection. Track results in array, count resolved count. Tests Promise mechanics.

16
Deep Clone — Deep clone an object handling all types.P2
Basic Answer

Handle: primitives (return directly), arrays (map with recursive clone), objects (Object.keys + recursive), Dates (new Date(date)), Maps (new Map), Sets (new Set), circular refs (WeakMap cache). Tests recursion and edge case thinking.

17
Deep Equal — Deep equality with helpful error messages.P2
Basic Answer

Compare primitives (Object.is), arrays (same length, recursive elements), objects (same keys, recursive values). Handle edge cases: dates, null, circular refs. Tests recursion and attention to detail.

18
JSON Validator — Check if string is valid JSON.P2
Basic Answer

try { JSON.parse(str); return true; } catch { return false; }. Tests understanding that the simplest solution is often best. Discuss trade-offs of manual parsing vs try/catch.

19
Flatten Nested Array — Recursive flatten with depth control.P2
Basic Answer

Use reduce + concat recursively. arr.flat(Infinity) for simple case. Implement custom: iterate, if item is array, recursively flatten and concat. Tests recursion.

20
Implement Retry Logic — Retry async function with backoff.P2
Basic Answer

async function retry(fn, maxRetries, delay) { for (let i = 0; i <= maxRetries; i++) { try { return await fn(); } catch (e) { if (i === maxRetries) throw e; await new Promise(r => setTimeout(r, delay * Math.pow(2, i))); } } }. Tests async, error handling, and backoff patterns.

21
Implement Array.prototype.map — Polyfill.P2
Basic Answer

Array.prototype.myMap = function(cb, thisArg) { const result = new Array(this.length); for (let i = 0; i < this.length; i++) { if (i in this) result[i] = cb.call(thisArg, this[i], i, this); } return result; }. Tests understanding of this, sparse arrays, and Array methods.

22
Implement Function.prototype.bind.P2
Basic Answer

Function.prototype.myBind = function(context, ...args) { const fn = this; return function(...innerArgs) { return fn.apply(context, [...args, ...innerArgs]); } }. Tests this binding, spread, and closures.

23
LRU Cache — Design with O(1) get and put.P2
Basic Answer

Use Map (preserves insertion order in JS) — on get, delete and re-set (moves to end). On put, if exists delete, add, if over capacity delete oldest (Map.keys().next().value). O(1) operations. Tests data structure knowledge.

24
Event Emitter — Implement publish/subscribe pattern.P2
Basic Answer

class EventEmitter { constructor() { this.events = {}; } on(event, fn) { (this.events[event] ||= []).push(fn); } emit(event, ...args) { (this.events[event] || []).forEach(fn => fn(...args)); } off(event, fn) { this.events[event] = this.events[event]?.filter(f => f !== fn); } }. Tests class and event-driven patterns.

🔵 P3 — Hard Problems

25
Group Anagrams — Group strings that are anagrams.P3
Basic Answer

Map of sorted-string → array of original strings. O(n * k log k) or O(n * k) with character count key. Tests hash map and sorting.

26
Product of Array Except Self — Without division.P3
Basic Answer

Left products array, right products array. Multiply element-wise. O(n) time, O(1) extra (excluding output). Tests prefix/suffix pattern and space optimization.

27
Serialize/Deserialize a Binary Tree.P3
Basic Answer

DFS with null markers ("null" for missing nodes). Preorder traversal. Tests tree traversal and recursive thinking.

28
Design a Rate Limiter (Token Bucket).P3
Basic Answer

Track last request time and count. On each request, calculate elapsed tokens (time * rate), cap at max, decrement if available. Tests math, edge cases, and clean API design.

29
Memoize Function — Generic memoization with cache key.P3
Basic Answer

function memoize(fn) { const cache = new Map(); return function(...args) { const key = JSON.stringify(args); if (cache.has(key)) return cache.get(key); const result = fn.apply(this, args); cache.set(key, result); return result; } }. Tests closures, caching strategies, and edge cases (Symbols, circular refs).

30
Implement a Simple Virtual DOM.P3
Basic Answer

Create element objects ({ type, props, children }), render to real DOM, diff two virtual trees (compare type, props, children recursively). Patch differences. This is the core of React! Tests deep understanding of DOM and reconciliation.
