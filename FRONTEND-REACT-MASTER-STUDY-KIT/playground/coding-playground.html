1. Array.map() Questions

Transform arrays — every React developer uses map() daily for rendering lists. Master it through these interview questions.

1
Basic map — Double each number
map easy
✅ Expected Output
Original: [1, 2, 3, 4, 5]
Doubled: [2, 4, 6, 8, 10]
Why? map() creates a new array by calling the callback on every element. The original array is never mutated — this is a key principle of immutability in React.
2
Extract property from array of objects
map easy
✅ Expected Output
Names: ["Alice", "Bob", "Charlie"]
Why? This is exactly how you extract data for rendering in React — users.map(u => <li>{u.name}</li>).
3
Using index in map — Add numbering
map medium
✅ Expected Output
["1. Apple", "2. Banana", "3. Cherry"]
Why? map() passes three args: (element, index, array). The second argument index is commonly used as React keys and numbering.
4
What does map return for empty slots?
map medium
✅ Expected Output
Result: [2, empty, 6]
Length: 3
Has index 1: false
Key insight: map() preserves sparse arrays — it skips empty slots entirely (the callback is NOT called for missing elements). This is critical when implementing polyfills. React handles sparse arrays by skipping empty slots too.
5
What does this map + parseInt output?
map medium
✅ Expected Output
[1, NaN, NaN]
Classic interview trap! parseInt takes (string, radix) as parameters. map() passes (element, index, array). So:
parseInt("1", 0) → 1 (radix 0 = auto-detect)
parseInt("2", 1) → NaN (radix 1 is invalid)
parseInt("3", 2) → NaN (binary has no digit 3)
Fix: .map(num => parseInt(num)) or .map(Number)
6
Deep transform — Nested object mapping
map hard
✅ Expected Output
[{key: 1, label: "ALICE", selected: false}, {key: 2, label: "BOB", selected: false}]
Why? Returning an object from map() requires wrapping in () so the parser doesn't confuse {} with a function body. This pattern is used daily in React for transforming API data into component props.
7
React keys — What's wrong with using index as key?
map easy
✅ Key Insight
Using index as key 👉 causes re-render bugs when items are reordered, added at the beginning, or removed.
React uses keys to identify elements — index is unstable and can cause incorrect component state reuse.
Solution: Use stable unique IDs (item.id), or combine with a text field if no IDs exist. The key prop tells React which items changed, were added, or removed.

2. Array.filter() Questions

Filter arrays — test your understanding of predicate functions, truthy/falsy values, and immutability.

1
Basic filter — Get even numbers
filter easy
✅ Expected Output
Original: [1, 2, 3, 4, 5, 6]
Evens: [2, 4, 6]
Why? filter() returns a new array with elements where the callback returns a truthy value. Original array is unchanged.
2
Filter out falsy values — what's the result?
filter easy
✅ Expected Output
Truthy values: [1, "hello", true]
Length: 3
Common pattern! filter(Boolean) removes all falsy values: 0, "", false, null, undefined, NaN. This is used frequently for cleaning up data before rendering. Note: empty arrays [] and empty objects {} are truthy!
3
Filter by property — Active users only
filter medium
✅ Expected Output
Active: ["Alice", "Charlie"]
Why? u.active returns the boolean value directly. If active is true, filter keeps the element. This is the exact pattern used for filtering lists in React components.
4
Remove duplicates using filter
filter medium
✅ Expected Output
Original: [1, 2, 2, 3, 4, 4, 5]
Unique: [1, 2, 3, 4, 5]
Clever trick! indexOf returns the first occurrence. If the current index matches, it's the first time we've seen that value. If not, it's a duplicate and gets filtered out. Downside: O(n²) — use Set for larger arrays.
5
What does filter return for an empty array?
filter hard
✅ Expected Output
Empty array: []
No matches: []
Truthy filter: [1, 2, 3]
Key insight: filter() always returns a new array, even if empty or if it contains all elements! result1 !== result2 (different references). This means if (arr.filter(...).length) is safer than if (arr.filter(...)) since empty arrays are truthy.
6
Chaining filter + map — Performance trap
filter map hard
✅ Expected Output
Chained result: [20, 40, 60, 80, 100]
Single-pass result: [20, 40, 60, 80, 100]
Performance insight: Chaining filter() + map() iterates the array twice. For large datasets (10,000+ items), reduce() in a single pass can be 2x faster. However, chaining is more readable — optimize only when profiling shows it matters.

3. Array.reduce() Questions

The most powerful array method — master reduce and you can implement any data transformation.

1
Basic reduce — Sum of numbers
reduce easy
✅ Expected Output
Sum: 15
Why? reduce takes a callback (accumulator, currentValue) and an initial value. Each iteration adds the current number to the accumulator. 0 is the initial value.
2
Reduce without initial value — what happens?
reduce medium
✅ Expected Output
Sum without initial: 15
Key difference: Without initial value, reduce() uses the first element as the accumulator and starts from index 1. Same result here, but dangerous for empty arrays! [].reduce((a,c) => a + c) throws TypeError: Reduce of empty array with no initial value. Always provide an initial value!
3
Flatten array using reduce
reduce medium
✅ Expected Output
Flattened: [1, 2, 3, 4, 5, 6]
Why? Each iteration concats the current sub-array to the accumulator. This is the classic reduce pattern for flattening — also works with [...acc, ...cur] (spread). For deeply nested arrays, use recursion or Array.flat(Infinity).
4
Group by property using reduce
reduce hard
✅ Expected Output
{"25": ["Alice", "Charlie"], "30": ["Bob", "Diana"], "35": ["Eve"]}
Powerful pattern! This grouping pattern is used in data visualization, category filtering, and dashboard widgets. The expression (acc[key] = acc[key] || []).push(value) initializes the array if it doesn't exist, then pushes the value.
5
Count occurrences using reduce
reduce hard
✅ Expected Output
{apple: 3, banana: 2, orange: 1}
Common interview question! (acc[fruit] || 0) + 1 safely handles first occurrence (undefined → 0). This pattern is useful for frequency analysis, vote counting, and data aggregation.
6
Implement map using reduce
reduce map medium
✅ Expected Output
Doubled (via reduce): [2, 4, 6, 8, 10]
Evens (via reduce): [2, 4]
Why this matters: This shows that reduce() is a general-purpose iteration method. Both map() and filter() can be implemented with reduce(). Understanding this proves mastery of the method.

4. console.log() Output Prediction

Tricky JavaScript output questions — common interview traps involving hoisting, closures, event loop, scoping, and coercion.

1
Hoisting with var
console medium
✅ Expected Output
undefined
5
Hoisting with var: var declarations are hoisted to the top of their scope and initialized with undefined. So console.log(a) sees undefined, not an error. After assignment, a becomes 5. With let/const, this would throw ReferenceError (Temporal Dead Zone).
2
Temporal Dead Zone with let
console medium
✅ Expected Output
ReferenceError: Cannot access 'b' before initialization
Temporal Dead Zone: let and const are hoisted but NOT initialized. Accessing them before declaration causes a ReferenceError. This is a common interview question to test ES6 scoping knowledge.
3
Classic closure loop — setTimeout + var
console hard
✅ Expected Output
3
3
3 (three times)
Classic closure trap! var is function-scoped — all three closures share the same i. By the time setTimeout fires, the loop has finished and i = 3.
Fix 1: Use let i (block-scoped, creates new binding each iteration)
Fix 2: for (var i = 0; i < 3; i++) { (j => setTimeout(() => console.log(j), 100))(i); } (IIFE)
4
Closure loop fixed with let
console medium
✅ Expected Output
0
1
2
Why it works with let: let is block-scoped. Each iteration creates a new binding for i. The closure captures the correct value. This is the simplest fix — always prefer let/const over var.
5
Event loop — setTimeout, Promises, and microtasks
console hard
✅ Expected Output
1
4
3
2
Event Loop order:
1. Synchronous code first: 1, then 4
2. Microtasks (Promise.then) before macrotasks (setTimeout): 3
3. Macrotasks: 2
Priority: Sync → Microtasks (Promises, queueMicrotask) → Animation frames → Macrotasks (setTimeout, setInterval, I/O)
6
Type coercion — == vs ===
console medium
✅ Expected Output
true (loose equality coerces string to number)
false (strict equality — different types)
true (false → 0, "0" → 0)
false
true (special case: null == undefined)
false
true ([] → "" → 0, false → 0)
true ([1] → "1" → 1, true → 1)
The Golden Rule: == performs type coercion before comparing. === requires same type AND value. Always prefer === unless you specifically need coercion (known as "the crockford rule"). Abstract Equality Comparison algorithm is a common interview deep-dive topic.
7
The 'this' keyword — implicit vs explicit binding
console hard
✅ Expected Output
Regular: MyObject
Arrow: undefined (or global object in non-strict)
undefined (or global)
Bound!
this binding rules:
1. Regular function called as method → this is the object
2. Arrow function → this is from outer (lexical) scope, not the object
3. Unbound reference → this is undefined (strict) or global (non-strict)
4. bind/call/apply → explicitly sets this
8
NaN behavior — typeof and equality
console medium
✅ Expected Output
number (typeof NaN is 'number'!)
false (NaN is the only value !== itself)
false
true (Object.is treats NaN correctly)
true (isNaN coerces to number first)
false (Number.isNaN doesn't coerce — "hello" is not NaN)
NaN quirks: Despite meaning "Not a Number", typeof NaN === "number". NaN is never equal to anything, including itself — NaN !== NaN is the only self-inequality in JS. Use Number.isNaN() for reliable checking.
9
Array and Object coercion
console medium
✅ Expected Output
"" (empty string)
"[object Object]"
0 (or "[object Object]" in some engines — depends on context)
"1,23,4"
Coercion: + operator treats objects as strings (calls toString()).
[] → "", {} → "[object Object]", [1,2] → "1,2".
[] + [] = "" + "" = ""
[] + {} = "" + "[object Object]" = "[object Object]"
[1,2] + [3,4] = "1,2" + "3,4" = "1,23,4"
10
Spread operator — shallow copy
console easy
✅ Expected Output
original.a: 1
copy.a: 99
original.b.c: 999 (mutated!)
copy.b.c: 999
Shallow copy! Spread is shallow — nested objects/arrays are still referenced, not cloned. copy.b points to the same object as original.b. For deep cloning, use structuredClone(), JSON.parse(JSON.stringify()), or a deep clone utility.
11
Destructuring with default values
console medium
✅ Expected Output
a: 1
b: null (default NOT used — value is null, not undefined)
c: 30 (default IS used — value is undefined)
d: 40 (default IS used — property doesn't exist)
Key insight: Default values in destructuring only apply when the value is undefined. null is a defined value and will NOT trigger the default. This is a very common interview trap!
12
Promise chaining — resolve values
console hard
✅ Expected Output
then1: 1
then2: 2
then3: 3
catch: Boom!
then4: recovered
Promise chaining rules:
1. Each .then() receives the resolved value from the previous step
2. Returning a value from .then() wraps it in a resolved promise
3. Returning Promise.resolve(x) is the same as returning x
4. throw in .then() jumps to the nearest .catch()
5. .catch() can recover — whatever it returns continues down the chain
13
async/await execution order
console hard
✅ Expected Output
script start
async1 start
async2
promise1
script end
async1 end
promise2
setTimeout
Event Loop deep-dive:
1. Sync: script start, async1 start, async2, promise1, script end
2. await suspends execution — the rest of async1 becomes a microtask
3. Microtasks: async1 end (from await), promise2 (from Promise.then)
4. Macrotask: setTimeout
Note: The await async2() is a microtask checkpoint, so async1 end runs before promise2.
14
Unary plus — type coercion to number
console easy
✅ Expected Output
123
1
0
0
NaN
0
NaN
Unary + operator: Converts operand to Number. Common shorthand for numeric coercion. +"123" is equivalent to Number("123"). Know the coercion table — null → 0, undefined → NaN, true → 1, false → 0, "" → 0.
15
Logical operators — short-circuit evaluation
console medium
✅ Expected Output
"hello" (first truthy)
"default" (0 is falsy, so return second)
"fallback"
"empty" ("" is falsy)
"world" (&& returns last truthy)
0 (first falsy — short-circuits)
"yes"
false
Short-circuit rules:
a || b — returns first truthy value, or last value if all falsy
a && b — returns first falsy value, or last value if all truthy
These are NOT boolean operators! They return the actual value, not true/false. This is why you can use condition && <Component /> in React.

5. JavaScript Array Fundamentals

18 essential array questions covering creation, manipulation, iteration, and common interview pitfalls.

1
Create an Array — in at least 3 methods
arrays easy
✅ Key Insight
5 different ways to create arrays. The important distinction: new Array(3) creates a sparse array of length 3 with empty slots (not [3]), while Array.of(3) creates [3].
Remember: new Array(n) creates empty slots when passed a single number. Use Array.of() if you want [n]. Array.from() is great for generating sequences with a map function.
2
Copy an array using slice() and for loop
arrays easy
✅ Key Insight
All three methods create shallow copies. For primitive arrays, this behaves like a deep copy. For nested objects — they still share references. slice() and [...arr] are the most common methods.
Interview tip: array.slice() is the classic way to copy an array. Modern code uses spread [...array]. Both are shallow — use structuredClone() for deep cloning.
3
Empty an array — multiple methods
arrays easy
✅ Key Insight
array.length = 0, splice(), and pop() loop all mutate the original array. Re-assigning array = [] creates a new reference — other references to the old array remain unchanged.
Important distinction: If you have const arr = [1,2,3]; const ref = arr; arr = [] — ref still points to [1,2,3]! Use arr.length = 0 to clear the original reference.
4
What type is an Array?
arrays easy
✅ Expected Output
"object"
Why? Arrays are objects in JavaScript. typeof returns "object" for arrays. To check if something is an array, use Array.isArray(arr). The typeof operator is useless for distinguishing arrays from other objects.
5
How to check if something is an Array?
arrays easy
✅ Key Insight
Array.isArray() is the only reliable way. typeof arr === "object" is true but doesn't distinguish from other objects. instanceof Array works across frames (unlike instanceof).
Why not instanceof? arr instanceof Array fails across different execution contexts (iframes, windows). Array.isArray() is safe across all contexts.
6
Add an item to the end of an array
arrays easy
✅ Key Insight
push() mutates the original. concat() and [...arr, item] return new arrays (immutable). In React, prefer immutable methods to avoid accidental mutations.
React rule: Never use push() in React state — use setState(prev => [...prev, newItem]) to maintain immutability.
7
Find index of an element
arrays easy
✅ Expected Output
3
indexOf() returns the first index at which a given element is found. Array indices start at 0, so "d" is at index 3.
8
What is returned when indexOf doesn't find the element?
arrays easy
✅ Expected Output
-1
indexOf() returns -1 when the element is not found. Always check if (index !== -1) before using the result — 0 is a valid index but falsy in a boolean context, so if (index) fails for index 0!
9
Check if "milk" exists, find index, and remove it
arrays medium
✅ Expected Output
Has milk? true
Milk index: 0
After removing milk: ["bread", "sugar"]
Pattern: includes() checks existence, indexOf() finds position, splice() removes by index. splice mutates the original — always use the -1 guard check!
10
Add code to find index of 'milk' and remove it
arrays easy
✅ Expected Output
["bread", "sugar"]
indexOf() finds the position, splice(index, 1) removes that item. Always check index !== -1 before splicing — the -1 guard is critical!
11
List the ways to loop over an array
arrays medium
✅ Key Insight
forEach() cannot be broken/stopped early. Use for...of with break/continue if you need early exit. for...of also works best with async/await — forEach() fires all callbacks immediately without waiting.
Sequence matters: for...of waits for each await sequentially. forEach() is synchronous-only — use it for operations without async side effects.
12
Sort numbers — Ascending & Descending (the default sort trap)
arrays hard
✅ Expected Output
Default sort: [1, 12, 2, 23, 234, 33, 5, 7, 77, 99]
Ascending: [1, 2, 5, 7, 12, 23, 33, 77, 99, 234]
Descending: [234, 99, 77, 33, 23, 12, 7, 5, 2, 1]
Classic JavaScript trap! sort() converts elements to strings and compares their UTF-16 values. "12" < "2" because "1" < "2". Always pass a compare function for numeric sorting! a - b for ascending, b - a for descending.
13
Sort strings alphabetically
arrays easy
✅ Expected Output
A-Z: ["a", "e", "y", "z"]
Z-A: ["z", "y", "e", "a"]
Locale sort: ["apple", "banana", "café", "résumé"]
Tip: Default string sort works as expected for simple letters. For international text with accents (é, ü), use localeCompare() for correct ordering. sort().reverse() chains nicely for descending order.
14
Array length — tricky cases
arrays medium
✅ Expected Output
arr1 length: 3
arr2 length: 3
arr3 length: 5
arr4 length: 2
Important distinctions:
arr1 and arr2 have length 3 but are sparse — they have empty slots (no properties at indices 0, 1, 2).
arr4.length is 2 because it has 2 elements (both are sub-arrays). Multi-dimensional array length only counts the first dimension.
15
splice vs slice — What's the difference?
arrays medium
✅ Expected Output
slice(1,3): ["one", "two"]
Original after slice: ["zero", "one", "two", "three"] (unchanged!)
splice(1,3): ["john", "peter", "karen"]
Original after splice: ["jason"]
Key difference:
slice() 📗 — does NOT mutate, returns subset (end index is exclusive)
splice() ✂️ — DOES mutate, removes/replaces items
Mnemonic: slice is like cutting a piece of cake (original stays), splice is like splicing rope (original is altered).
16
shift and unshift — track the changes
arrays hard
✅ Expected Output
b: [22, 1]
c: [1]
d: [3, [4,5], 1]
e: [[4,5], 1]
f: [1]
g: []
Trace:
1. unshift(1) → [1]
2. unshift(22) → [22, 1] ← b
3. shift() removes 22 → [1] ← c
4. unshift(3, [4,5]) → [3, [4,5], 1] ← d
5. shift() removes 3 → [[4,5], 1] ← e
6. shift() removes [4,5] → [1] ← f
7. shift() removes 1 → [] ← g
Note: unshift adds to the beginning, shift removes from the beginning.
17
Using reduce to sum all numbers
reduce arrays easy
✅ Expected Output
Sum: 21
Total (for loop): 21
1 + 2 + 3 + 4 + 5 + 6 = 21
Reduce is the functional way to aggregate. Always provide an initial value (0 for sums) to avoid errors on empty arrays.
18
Flatten 2D array using reduce
reduce arrays medium
✅ Expected Output
Flattened: [0, 1, 2, 3, 4, 5]
Using flat(): [0, 1, 2, 3, 4, 5]
Deep flat(1): [[1,2], [3,4], [5,6]]
Deep flat(2): [1, 2, 3, 4, 5, 6]
Deep flat(Infinity): [1, 2, 3, 4, 5, 6]
Flattening methods:
1. reduce + concat — classic interview answer (works for 1 level)
2. Array.flat(n) — modern, specify depth
3. Array.flat(Infinity) — completely flatten regardless of nesting
The reduce approach proves you understand how to manipulate arrays programmatically.
