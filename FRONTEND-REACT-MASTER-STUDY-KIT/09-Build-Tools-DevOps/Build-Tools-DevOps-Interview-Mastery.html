P1 — Frequently Asked Questions

1
Compare Vite vs Webpack. Why is Vite preferred for React projects?P1
Basic Answer

Vite: fast dev server (native ESM, no bundling needed), HMR via native ESM, Rollup-based production builds, simpler config, fast cold start. Webpack: mature, extensive plugin ecosystem, more configuration, slower dev server (needs to bundle everything). Vite is the recommended build tool for new React projects (CRA is deprecated). For complex configs (Module Federation), Webpack is still needed.

2
What is Babel? How does it fit into the React build pipeline?P1
Basic Answer

Babel transpiles modern JS/TS and JSX to browser-compatible JS. React pipeline: JSX → Babel (React JSX transform) → browser-compatible JS. Presets: @babel/preset-env (ES features), @babel/preset-react (JSX), @babel/preset-typescript. With Vite, SWC (Rust-based) is often used instead of Babel for faster compilation.

3
How do you configure ESLint and Prettier for a React project?P1
Basic Answer

ESLint: static analysis for code quality. eslint-plugin-react (React rules), eslint-plugin-react-hooks (hooks rules — must have!), @typescript-eslint/parser, eslint-config-next (Next.js). Prettier: code formatting (opinionated). Integration: eslint-config-prettier (disables ESLint rules that conflict with Prettier), eslint-plugin-prettier (runs Prettier as ESLint rule). Husky + lint-staged for pre-commit hooks.

4
Explain code splitting and lazy loading in React.P1
Basic Answer

Code splitting breaks the bundle into smaller chunks loaded on demand. React.lazy(() => import('./Component')) + Suspense. Route-based splitting (most effective): load components per route. Dynamic imports for heavy libraries (load when needed). Webpack/Vite create separate chunks automatically. Benefits: smaller initial bundle, faster time-to-interactive.

5
How do you set up CI/CD for a React application?P1
Basic Answer

GitHub Actions: popular choice for React projects. Pipeline stages: lint → type-check → test → build → deploy. Example: GitHub Actions + Vercel (auto-deploy on main). For self-hosted: Dockerize app, push to registry, deploy to Kubernetes/ECS. Environment management: .env.production, secrets in CI. Preview deployments for PRs (Vercel/Netlify).

6
Docker for frontend development — how would you Dockerize a React app?P1
Basic Answer

Multi-stage Dockerfile: Stage 1 (build) — Node image, install deps, build. Stage 2 (production) — Nginx image, copy build output, serve static files. Use .dockerignore to exclude node_modules. For development: Docker Compose with volumes for hot reload. Example: FROM node:18 AS builder → FROM nginx:alpine.

7
What is Module Federation? When would you use it?P1
Basic Answer

Module Federation (Webpack 5) enables micro-frontends by allowing separate builds to share code at runtime. One app can expose/consume components from another app. Use cases: enterprise apps with multiple teams, shared component libraries, gradual migration between frameworks. Downside: increased complexity, shared dependency versioning. Alternatives: Iframes, NPM packages, Native Federation (for Vite/Rollup).

8
What are monorepos? Compare Turborepo vs Nx.P1
Basic Answer

Monorepo: multiple projects in one repository. Sharing code, unified CI, atomic commits. Turborepo (Vercel): task orchestration, caching, parallel execution. Nx (Nrwl): more features (generators, affected commands, dependency graph). Both use workspaces (npm/pnpm/yarn). Choose Turborepo for simpler needs, Nx for complex enterprise setups.

9
How do you analyze and optimize bundle size?P1
Basic Answer

Tools: vite-bundle-visualizer, webpack-bundle-analyzer, source-map-explorer. Strategies: code splitting (route-based), tree-shaking (remove unused exports), dynamic imports for heavy libraries, analyze dependency weight (moment.js → date-fns/luxon), reduce lodash to cherry-picked imports, use smaller alternatives (zustand instead of redux), compress images (next/image), polyfills sizing.

10
What is SWC? How does it compare to Babel for React projects?P1
Basic Answer

SWC (Speedy Web Compiler) is a Rust-based compiler for JS/TS. 20x faster than Babel. Used by Next.js (since v12) and Vite. Transforms JSX, TypeScript, and modern JS. @swc/core + @swc/jest for Jest transform. Downside: fewer plugins than Babel, but covers the essential needs for React projects.

P2 — Commonly Asked Questions

11
How does tree-shaking work in modern bundlers?P2
Basic Answer

Tree-shaking removes unused exports from bundles. Requires ES modules (static imports, not CommonJS). Webpack/Rollup analyze import/export relationships and eliminate dead code. Side effects matter — "sideEffects": false in package.json tells bundlers it's safe to tree-shake. Some libraries (lodash, moment.js) are hard to tree-shake due to side effects.

12
What are the different deployment strategies for React apps?P2
Basic Answer

Static hosting (SPA): Vercel, Netlify, S3 + CloudFront, GitHub Pages. SSR: Vercel (Next.js), AWS Lambda, Node.js server. Containerized: Docker + ECS/Kubernetes. Edge: Vercel Edge Functions, Cloudflare Workers. For SPAs: configure redirect rules (all routes → index.html). For Next.js: Vercel is the most seamless deployment experience.

13
How do you manage environment variables in React?P2
Basic Answer

.env files: .env.local (local), .env.production. Vite: VITE_ prefix. CRA: REACT_APP_ prefix. Next.js: NEXT_PUBLIC_ prefix for client, no prefix for server-only. Never commit .env.local. Use CI secrets for production values. Runtime env vars: Next.js supports runtime configuration.

14
What is the difference between npm and yarn and pnpm?P2
Basic Answer

npm: default, improved significantly (v7+). yarn: deterministic lockfile, speed improvements. pnpm: fastest, disk-efficient (hard links), strict dependency resolution (no phantom deps). pnpm is gaining popularity for monorepos and large projects. npm is sufficient for most projects.

15
How does HMR (Hot Module Replacement) work in React?P2
Basic Answer

HMR updates modules in the browser without full page reload. Preserves component state. Vite: native ESM-based HMR (file change → send updated module → React Refresh updates component). Webpack: HMR runtime updates chunk. React Fast Refresh (React Refresh plugin) is the modern HMR implementation — preserves state even when components change.

16
How do you set up a monorepo for multiple React apps?P2
Basic Answer

pnpm workspaces or npm workspaces. Turborepo/Nx for task orchestration. Structure: apps/web, apps/docs, packages/ui, packages/config. Shared TypeScript config, ESLint config, and component library. Turborepo: turbo.json for pipeline config. Caching for faster CI. Versioning with Changesets.

17
What is esbuild and why is it fast?P2
Basic Answer

esbuild is a Go-based bundler/compiler. 10-100x faster than JS-based tools. Used by Vite for dependency pre-bundling (node_modules) and by many tools under the hood. Supports JSX, TS, CSS, JSON. Limited plugin ecosystem compared to Webpack but covers most needs.

18
How do you handle polyfills for older browsers?P2
Basic Answer

Babel @babel/preset-env with core-js for automatic polyfilling based on browser targets (browserslist). useBuiltIns: 'usage' — only include polyfills for used features. Modern approach: serve modern code to modern browsers (module/nomodule pattern). Next.js handles this automatically with browser targets.

19
What is the browserslist config and how does it affect builds?P2
Basic Answer

browserslist defines target browsers. Used by Babel (@babel/preset-env), Autoprefixer, PostCSS, and other tools. Example: "> 0.5%, last 2 versions, not dead". Affects: which syntax is transpiled, which polyfills are added, CSS vendor prefixes. Broader support → larger bundle. Narrower support → smaller bundle.

20
How do you optimize images in a React build pipeline?P2
Basic Answer

Next.js next/image for automatic optimization. Vite: vite-plugin-imagemin for build-time compression. Webpack: image-webpack-loader. Techniques: WebP/AVIF format, responsive sizes, lazy loading (loading="lazy"), CDN for images (Cloudinary, Imgix), CSS sprites for icons. Modern approach: use CDN image services that handle optimization and format negotiation.

P3 — Occasionally Asked (Valuable Context)

21
What is postcss-preset-env and how does it differ from Babel?P3
Basic Answer

PostCSS transforms CSS with plugins. postcss-preset-env enables future CSS syntax today (custom properties, nesting, media query ranges). Babel handles JS/TS. PostCSS handles CSS. Tailwind CSS is a PostCSS plugin. Autoprefixer (PostCSS plugin) adds vendor prefixes based on browserslist.

22
What is the module/nomodule pattern?P3
Basic Answer

Serves modern (ES module) code to modern browsers and legacy (script) code to older browsers. . Modern browsers execute module scripts, ignore nomodule. Legacy browsers ignore module, execute nomodule. Reduces bundle size for 90%+ of users. Next.js uses this pattern.

23
How do you implement feature flags in React CI/CD?P3
Basic Answer

Server-side: LaunchDarkly, Flagsmith, Split.io. Client-side: environment variables, custom FeatureFlagProvider + context. Build-time flags: import.meta.env.VITE_FEATURE_X (Vite), process.env.FEATURE_X (Webpack). Runtime flags: API-driven (fetch flags on app load). Canary releases with deployment platform (Vercel).

24
How does Vite handle dependency pre-bundling?P3
Basic Answer

Vite pre-bundles dependencies (node_modules) using esbuild at startup. Converts CommonJS dependencies to ESM. Caches results in node_modules/.vite. Speeds up dev server — no need to process dependencies on each file change. Re-runs when dependencies change (package-lock.json update). Configurable with optimizeDeps.

25
What is the future of frontend build tools?P3
Basic Answer

Turbopack (Rust-based, Vercel) — potential successor to Webpack for Next.js (still beta). Rspack (Rust-based, ByteDance) — Webpack-compatible, faster. Bun — all-in-one runtime/bundler/package manager. Rome (Rust-based) — formatter/linter/bundler. Trend: tools moving from JS to Rust/Go for performance. Vite is current leader for non-Next.js projects.
