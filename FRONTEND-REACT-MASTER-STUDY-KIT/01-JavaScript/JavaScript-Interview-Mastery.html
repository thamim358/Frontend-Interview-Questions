P1 — Frequently Asked Questions

You MUST know these. They are asked in almost every interview and technical round.

1
Explain the difference between var, let, and const with hoisting and scope. Advanced P1
Basic Answer

var works inside the whole function. let and const work only inside the nearest {} block. var is hoisted with undefined. let and const are hoisted but not ready to use (Temporal Dead Zone). const cannot be reassigned, but you can change object properties inside it. Rule: always use const first in React code. Use let only if you need to reassign. Never use var.

Real Interview Answer

"var works across the whole function and gets hoisted with undefined. let and const only work inside the block they're declared in. They are hoisted too but stuck in the Temporal Dead Zone until you reach them. In my React code, I use const for everything — components, hooks, state values. I use let only for loop counters or conditional reassignments."

2
What is a closure? Give real-world examples of closures in React. Advanced P1
Basic Answer

A closure is a function that remembers variables from where it was created, even after that outer function has finished running. This lets you keep data private and save state. In React, closures are everywhere: useEffect callbacks capture state values, event handlers close over props, and custom hooks create private state through closures.

Real Interview Answer

"Closures are fundamental to React. Every useEffect callback is a closure over the component's props and state at render time. That's why you need the dependency array — to ensure the closure always has the latest values. Custom hooks create private state through closures, similar to how useState preserves state across renders. One common bug: stale closures in useEffect when you forget to include dependencies, or in setTimeout callbacks inside useEffect."

3
How does the event loop work? Explain call stack, microtask queue, and macrotask queue. Expert P1
Basic Answer

The event loop is how JavaScript handles async tasks with one thread. It checks if the call stack is empty. If empty, it takes tasks from the microtask queue (high priority — Promises) or macrotask queue (low priority — setTimeout). This is why Promises resolve before setTimeout callbacks. In React, understanding the event loop helps debug why state updates might seem delayed — setState batches updates using microtasks.

Real Interview Answer

"The event loop runs sync code first, then drains ALL microtasks (Promise resolutions, queueMicrotask), then processes ONE macrotask (setTimeout, setInterval), and repeats. This matters in React because state updates are batched using microtasks — multiple setState calls in a synchronous block are batched into a single re-render. Also, useEffect callbacks run after the render is committed to the DOM, which relates to the event loop scheduling."

console.log(1); // Sync — runs 1st setTimeout(() => console.log(2), 0); // Macrotask — runs 4th Promise.resolve().then(() => console.log(3)); // Microtask — runs 3rd console.log(4); // Sync — runs 2nd // Output: 1, 4, 3, 2
4
Explain this binding — default, implicit, explicit, arrow function, and new binding.AdvancedP1
Basic Answer

this depends on HOW you call a function. Default: global object (undefined in strict mode). Implicit: object before the dot. Explicit: call/apply/bind. Arrow: inherits from surrounding scope. new: the new instance. In React classes (legacy), this binding was a common pain point. With hooks, arrow functions in components avoid this issues entirely.

Real Interview Answer

"In modern React with hooks, this is less of an issue since function components and hooks don't use it. But understanding this is still important for: event handlers in class components, implementing custom hooks that use method binding patterns, and understanding legacy React code. The key priority order: new > explicit > implicit > default. Arrow functions inherit this lexically."

5
What are Promises? Explain Promise.all, Promise.race, Promise.any, and Promise.allSettled.IntermediateP1
Basic Answer

A Promise represents a future value. States: pending, fulfilled, rejected. Promise.all — waits for ALL, fails fast on any rejection. Promise.race — returns first settled (resolve or reject). Promise.any — returns first SUCCESS, ignores failures. Promise.allSettled — waits for all, never fails, returns results array. In React, these are essential for data fetching patterns in useEffect and React Query.

// In a React component useEffect(() => { const fetchData = async () => { try { const [users, posts] = await Promise.all([ fetch('/api/users').then(r => r.json()), fetch('/api/posts').then(r => r.json()) ]); setUsers(users); setPosts(posts); } catch (error) { setError(error); } }; fetchData(); }, []);
6
How does async/await work? How is error handling different from .then()/.catch()?IntermediateP1
Basic Answer

async/await is syntactic sugar over Promises. An async function always returns a Promise. await pauses the function until the Promise resolves — but doesn't block the main thread. Use try/catch for error handling. In React, async/await is used in useEffect, event handlers, React Query, and Server Components (Next.js).

7
What is hoisting? How does it differ for var, let, const, and function declarations?IntermediateP1
Basic Answer

Hoisting means declarations are moved to the top during compilation. Function declarations are fully hoisted (can call before declaration). var is hoisted with undefined. let and const are hoisted but uninitialized (Temporal Dead Zone). In React, this affects component ordering — you can define helper components after they're used if they're function declarations.

8
What is prototypal inheritance? How does the prototype chain work?AdvancedP1
Basic Answer

Every object has a hidden link to its prototype. Property access checks the object, then the prototype chain until found or null. In React, understanding prototypes helps when extending class components (legacy) and understanding how React's synthetic event system works with event pooling (React 16 and earlier).

9
Explain map(), filter(), and reduce() with React examples.IntermediateP1
Basic Answer

map() transforms each element — used in React for rendering lists. filter() picks elements passing a test — used for filtering data before rendering. reduce() combines elements into one value — used for calculations and data transformations.

// React usage {items.map(item => ( <div key={item.id}>{item.name}</div> ))} // Filter before render const activeUsers = users.filter(u => u.isActive); const totalPrice = cart.reduce((sum, item) => sum + item.price, 0);
10
How do you deep clone an object? What are the limitations of JSON.parse(JSON.stringify())?IntermediateP1
Basic Answer

JSON.parse(JSON.stringify(obj)) drops functions, undefined, Symbols, BigInt, circular references, and converts Date to string. Use structuredClone() (Node 17+) for better deep cloning. In React, deep cloning is important for immutable state updates, especially when working with Redux or Context — you must create new references to trigger re-renders.

11
Explain event delegation, event bubbling, and event capturing in the DOM.IntermediateP1
Basic Answer

Events go through three phases: capturing (window → target), target, bubbling (target → window). React's synthetic event system wraps native events and provides cross-browser consistency. React 17+ no longer attaches event handlers to the document but to the root React container. Event delegation is useful for performance when rendering long lists.

12
What are arrow functions? How do they differ from regular functions regarding this, arguments, and constructors?IntermediateP1
Basic Answer

Arrow functions have no own this (inherit from scope), no arguments object, and can't be used as constructors. In React, arrow functions are used everywhere — component definitions, useEffect callbacks, event handlers. They're preferred because they don't bind their own this, avoiding common bugs in class components.

13
What is the difference between == and ===? When would you use each?BeginnerP1
Basic Answer

== coerces types before comparing. === compares both value and type. Always use === in React code. The only exception is checking both null and undefined: value == null. In React, use strict equality for dependency arrays, conditional rendering, and props comparison.

14
What is the spread operator vs rest operator? Give React examples.IntermediateP1
Basic Answer

Spread (...) spreads items out. Rest (...) collects items together. In React, spread is used everywhere: {...props} to pass props, {...prevState, newKey: value} for immutable state updates, [...prevArray, newItem] for adding to arrays. Rest is used for const { usedProp, ...restProps } = props.

15
Implement debounce and throttle. When would you use each in React?AdvancedP1
Basic Answer

Debounce: waits until calls stop before running. Use for search inputs, resizing events. Throttle: limits execution rate. Use for scroll handlers, infinite scroll, API rate limiting. In React, debounce is commonly used in custom hooks like useDebounce for search inputs to avoid excessive API calls.

// useDebounce hook function useDebounce(value: T, delay: number): T { const [debouncedValue, setDebouncedValue] = useState(value); useEffect(() => { const timer = setTimeout(() => setDebouncedValue(value), delay); return () => clearTimeout(timer); // Cleanup on unmount or value change }, [value, delay]); return debouncedValue; }
16
What is the nullish coalescing operator (??) vs OR (||)? Give React examples.IntermediateP1
Basic Answer

?? returns right side only when left is null or undefined. || returns right side for ALL falsy values (false, 0, ''). In React, use ?? for default values where 0 or '' are valid — like pagination (page count could be 0) or text input values.

// React examples const count = props.count ?? 0; // Correct: 0 is a valid count const count2 = props.count || 0; // Bug: replaces 0 with... 0 (same result here, but logical error) // Conditional rendering {name ?? 'Guest'} // Shows nothing if name is '' — correct {name || 'Guest'} // Shows 'Guest' if name is '' — might be wrong!
17
How does Object.freeze() differ from Object.seal()?AdvancedP1
Basic Answer

seal — no add/delete, can change values. freeze — no add/delete/change. Both are shallow. In React/Redux, Object.freeze is used in development mode to prevent accidental state mutations — Redux Toolkit uses Immer which makes this easier.

18
What is the difference between Map/WeakMap and Set/WeakSet?AdvancedP1
Basic Answer

WeakMap keys must be objects and are garbage-collected when the key is no longer referenced. In React, Map is useful for storing lookup tables for performance, and Set for tracking selected items in lists. WeakMap can be useful for caching DOM node metadata without preventing garbage collection.

19
What is NaN? How do you reliably check for it?IntermediateP1
Basic Answer

NaN is "Not-a-Number" and is the only value not equal to itself (NaN !== NaN is true). Use Number.isNaN() (not global isNaN() which coerces). In React forms, validate numeric inputs with Number.isNaN to detect invalid number conversions.

20
Explain call(), apply(), and bind(). Implement a bind() polyfill.AdvancedP1
Basic Answer

call — invoke with this + individual args. apply — invoke with this + args array. bind — returns new function with this permanently set. In React, bind was common in class components for event handlers. With hooks, this is less needed.

P2 — Commonly Asked Questions

Questions often asked in technical screens and mid-level interviews.

21
What are the primitive data types? How do they differ from reference types?BeginnerP2
Basic Answer

7 primitives: string, number, boolean, null, undefined, symbol, bigint. Stored by value. Reference types: objects, arrays, functions — stored by reference. In React, this is why state mutations don't trigger re-renders — you must create new references.

22
What are truthy and falsy values? How do they affect conditional rendering?BeginnerP2
Basic Answer

6 falsy values: false, 0, '', null, undefined, NaN. Everything else is truthy. In React conditional rendering: {count && } will render 0 if count is 0 — use {count > 0 && } or ternary instead.

23
What is the difference between null and undefined?BeginnerP2
Basic Answer

undefined means a variable exists but hasn't been assigned. null is intentionally set "nothing". In React, checking value == null catches both. When destructuring props with defaults, undefined triggers the default but null does not.

24
How do splice() and slice() differ? Why should you be careful with them in React?BeginnerP2
Basic Answer

slice() returns a new array (non-mutating). splice() changes the original array (mutating). In React state management, NEVER use splice() on state arrays — it mutates the original. Use slice() or spread operator for immutable updates.

25
What is JSON.stringify() and JSON.parse()? How are they used in React?BeginnerP2
Basic Answer

stringify converts object → JSON string. parse converts JSON string → object. Used in React for: API responses, localStorage persistence, dependency arrays for object comparison (though this is an anti-pattern), deep cloning, and debugging with console.log(JSON.stringify(obj, null, 2)).

26
How does try/catch/finally work? Give a React error boundary example.BeginnerP2
Basic Answer

try wraps risky code. catch handles errors. finally always runs. In React, try/catch works in event handlers and useEffect, but NOT during rendering — you need Error Boundaries for that. Async errors in useEffect should be caught with try/catch.

27
What is an IIFE? Why would you use one?IntermediateP2
Basic Answer

IIFE (Immediately Invoked Function Expression) runs immediately. Creates private scope. Less common in modern React with ES modules, but useful for: initializing state with computation, running async code at module level, or creating private variables.

28
What is currying? How is it useful in React?AdvancedP2
Basic Answer

Currying transforms f(a, b, c) into f(a)(b)(c). In React, useful for: HOCs (Higher-Order Components), custom hooks with configuration, event handlers with extra params: onClick={(e) => handleClick(id, e)}.

29
How do you remove duplicates from an array? Compare approaches.IntermediateP2
Basic Answer

For primitives: [...new Set(arr)]. For objects: use Map with unique key or filter + findIndex. In React, deduplication is common when normalizing API data or filtering out duplicates before rendering lists.

30
What is memoization? Implement a memoize function.AdvancedP2
Basic Answer

Memoization caches function results by arguments. In React, memoization is built-in with React.memo, useMemo, and useCallback. Key insight: these are NOT free — they add comparison overhead. Only use when the computation is expensive or prevents unnecessary re-renders.

31
How does forEach() differ from map()? When to use each in React?BeginnerP2
Basic Answer

forEach returns nothing (for side effects). map returns a new array (for transformations). In React, use map() for rendering lists (items.map(item => )). Use forEach for side effects like logging or triggering actions.

32
How does destructuring work? Give React patterns.IntermediateP2
Basic Answer

Destructuring unpacks values from arrays/objects. In React: function Component({ title, children, ...props }) — props destructuring, const { data, isLoading } = useQuery() — hook destructuring, const [count, setCount] = useState(0) — array destructuring from useState.

33
What are template literals? Give React/JSX examples.BeginnerP2
Basic Answer

Template literals use backticks with ${expr}. In React: dynamic class names `btn-${variant}`, dynamic URLs `/users/${id}`, multi-line strings, and tagged templates for CSS-in-JS (Styled Components).

34
What is the difference between local storage, session storage, and cookies?IntermediateP2
Basic Answer

Local storage: persists across tabs, until cleared. Session storage: per-tab, cleared on close. Cookies: sent with HTTP requests, max 4KB, can set expiry. In React, use localStorage for theme preferences, auth tokens (with caution), or cached data.

35
What is typeof? What is typeof null?BeginnerP2
Basic Answer

typeof null returns 'object' — a known JS bug. typeof [] returns 'object' — use Array.isArray(). In React prop validation (or TypeScript), be aware of these quirks when checking prop types dynamically.

36
What is a pure function vs impure function? Why does it matter for React?IntermediateP2
Basic Answer

Pure functions: same input → same output, no side effects. React components should be pure — given the same props, they should render the same output. Side effects go in useEffect. Redux reducers must be pure. React's performance optimizations (memoization) rely on purity.

37
How do you flatten a nested array? Use cases in React?IntermediateP2
Basic Answer

arr.flat(depth) or arr.flat(Infinity). In React, useful when normalizing nested API data, flattening comments/replies, or processing tree-structured data for flat lists.

38
What are generators? When would you use them?AdvancedP2
Basic Answer

Generators (function*) can pause/resume execution with yield. Less common in React but useful for: infinite loading with custom iteration, Redux Saga (legacy), generating unique IDs or test data.

39
How does short-circuit evaluation work with && and ||?BeginnerP2
Basic Answer

&& returns first falsy or last truthy. || returns first truthy or last falsy. In React: {isLoggedIn && } is a common conditional rendering pattern. BUT be careful: {count && } renders 0 when count is 0.

40
What is the difference between for...in and for...of?BeginnerP2
Basic Answer

for...in loops over keys (use for objects). for...of loops over values (use for arrays, strings, Maps). In React, prefer for...of for array iteration or .map() for rendering.

41
How does type coercion work? Give examples.IntermediateP2
Basic Answer

Implicit: '5' - 3 = 2, '5' + 3 = '53'. Explicit: Number('5'), String(5). In React, be careful with + for concatenation vs addition — especially with form values which are always strings.

42
What is Array.isArray() vs instanceof Array?IntermediateP2
Basic Answer

Array.isArray() is reliable across frames. instanceof Array can fail with cross-frame arrays. In React, when normalizing API data, use Array.isArray(data) to safely check response types.

43
How do you check if an object has a property (own vs inherited)?IntermediateP2
Basic Answer

Object.hasOwn(obj, 'key') — own properties only. 'key' in obj — own + inherited. In React, when validating API responses or config objects, use Object.hasOwn to check for specific fields.

44
What is the difference between undefined and null checks in React props?IntermediateP2
Basic Answer

React's default props only work for undefined, not null. Destructuring defaults: const { name = 'Guest' } = props only applies when props.name is undefined. Use ?? for null-safe defaults: const name = props.name ?? 'Guest'.

45
Explain reduce() for grouping, counting, and flattening in React.IntermediateP2
Basic Answer

reduce combines elements into one value. Use cases in React: grouping API data by category for organized rendering, counting status occurrences for dashboards, computing totals and aggregates, transforming arrays into lookup objects for O(1) access.

// Group by status for a Kanban board const grouped = tasks.reduce((acc, task) => { acc[task.status] = acc[task.status] || []; acc[task.status].push(task); return acc; }, {});

P3 — Occasionally Asked (Valuable Context)

Less common but useful for senior roles and deep-dive interview rounds.

46
What is strict mode and what does it prevent?IntermediateP3
Basic Answer

Strict mode ("use strict") catches common coding errors. Prevents: undeclared variables, deleting non-deletable properties, duplicate params, this being global, the with statement. React uses strict mode internally in development.

47
What are Symbols? Where are they useful in React?AdvancedP3
Basic Answer

Symbols are unique, immutable values used as property keys. React uses Symbol.for('react.element') internally for element identification. Useful for creating truly unique keys in complex data structures, avoiding property name collisions, and implementing custom iterators.

48
Explain the Proxy API. React/Immer use case?ExpertP3
Basic Answer

Proxy intercepts object operations (get, set, has). Immer uses Proxies extensively — when you write mutable-looking code inside produce(), Immer intercepts all mutations via Proxy and creates immutable snapshots. Vue 3 uses Proxies for reactivity. React could potentially use them for state management (MobX uses them).

49
How does Object.create() work? React use case?AdvancedP3
Basic Answer

Object.create(proto) creates a new object with a given prototype. Object.create(null) creates a prototype-less object — useful for dictionaries/safe maps. In React, useful for lookup tables in reducers or memoized selectors.

50
What is requestAnimationFrame? When would you use it in React?AdvancedP3
Basic Answer

requestAnimationFrame schedules code before the next browser repaint (~60fps). Useful for: animations, canvas rendering, scroll-based animations. In React, can be used with custom hooks for performant animations. Pauses when tab is inactive.

51
What are Web Workers? How would you use them in a React app?ExpertP3
Basic Answer

Web Workers run JS in a background thread with their own event loop. Can't access DOM. Use in React: heavy data processing (CSV parsing, data transformation), image manipulation, large list filtering, or running WebAssembly modules.

52
What is the Reflect API? How does it relate to Proxy?ExpertP3
Basic Answer

Reflect provides methods matching Proxy traps. Use inside Proxy traps to forward default behavior: return Reflect.get(target, prop, receiver). In React ecosystem, Immer uses Reflect/Proxy internally.

53
What is setTimeout vs setInterval? Pitfalls in React?BeginnerP3
Basic Answer

setTimeout runs once after delay. setInterval runs repeatedly. In React, always clean up timers in useEffect cleanup to prevent memory leaks and stale closures. Use refs to store timer IDs for proper cleanup.

54
What is the difference between deep and shallow comparison in React?IntermediateP3
Basic Answer

React uses shallow comparison by default (===) for props. React.memo does shallow comparison. useEffect dependencies use Object.is() (same as === except for NaN). Deep comparison is expensive — avoid it. Use immutable updates so shallow comparison works.

55
How does Error.captureStackTrace() work?ExpertP3
Basic Answer

V8 method that captures a stack trace at call time. Used by libraries like React and Redux to provide clean error stack traces that point to user code, not library internals. Important for understanding React error messages.
