P1 — Frequently Asked Questions

You MUST know these. They are asked in almost every Frontend interview and technical round.

1
Explain var, let, and const with hoisting and scope. Advanced P1
Quick Answer

Think of it like backpacks of different sizes: var is a leaky backpack — its contents spill outside blocks. let and const are sealed backpacks — what's inside stays inside the block.

var works across the whole function and gets hoisted with undefined. let and const only work inside one block {}. They're hoisted too but stuck in the Temporal Dead Zone until you reach their declaration. const can't be reassigned (but object properties can change).

// var — leaks outside blocks if (true) { var x = 5; } console.log(x); // 5 (leaked out!) // let — contained in block if (true) { let y = 5; } console.log(y); // ❌ Error // const — can't reassign, but can mutate objects const obj = { name: "John" }; obj.name = "Jane"; // ✅ OK // obj = {}; // ❌ Error
Say in Interview

"I use const for everything in React — components, hooks, state values. I use let only when I need to reassign (like loop counters). Never var. One thing to remember: all three are hoisted, but let and const are in the Temporal Dead Zone until you reach their declaration line."

2
What is a closure? Give React examples. Advanced P1
Quick Answer

Think of a closure as a "backpack." When you create a function inside another function, the inner function carries a backpack containing the outer function's variables — even after the outer function has finished running.

In React, closures are everywhere: every useEffect callback captures the state and props from that render. Every event handler closes over component values. Custom hooks create private state through closures. This is how React works under the hood.

// Counter using closure — keeps count private function createCounter() { let count = 0; return () => ++count; } const counter = createCounter(); counter(); // 1 counter(); // 2 // React — useEffect creates a closure over count function Counter() { const [count, setCount] = useState(0); useEffect(() => { const timer = setInterval(() => { console.log(count); // This 'count' is captured in a closure }, 1000); return () => clearInterval(timer); }, [count]); // ↑ count in deps ensures closure has latest value }
Say in Interview

"Closures are fundamental to React. Every single useEffect callback is a closure over the component's props and state at render time. That's why you need the dependency array — to ensure the closure always has the latest values. A common bug is a stale closure in useEffect or setTimeout when you forget to include dependencies."

3
Event Loop — call stack, microtask queue, macrotask queue? Expert P1
Quick Answer

Think of it as a restaurant kitchen: The call stack is the chef cooking one dish at a time. Microtasks are VIP orders that jump ahead of regular orders (macrotasks). The event loop is the manager checking: "Is the chef free? Great, bring in the next VIP order. Then bring one regular order."

Execution order: Run all sync code → Run ALL microtasks (Promise callbacks) → Run ONE macrotask (setTimeout, I/O) → Repeat.

console.log(1); // Sync — 1st setTimeout(() => console.log(2), 0); // Macrotask — 4th Promise.resolve().then(() => console.log(3)); // Microtask — 3rd console.log(4); // Sync — 2nd // Output: 1, 4, 3, 2
Say in Interview

"The event loop runs sync code first, drains ALL microtasks (Promise resolutions), then processes ONE macrotask. This matters in React because state updates are batched using microtasks — multiple setState calls in a synchronous block are batched into one re-render. Also, useEffect callbacks run after the render is committed to the DOM."

4
How does this work? Default, implicit, explicit, arrow, new? Advanced P1
Quick Answer

Memory trick: this depends on HOW you call a function, not where you write it (except arrow functions).

Priority order: new > explicit (call/apply/bind) > implicit (obj.method()) > default (global or undefined in strict). Arrow functions ignore all these — they inherit this from where they're defined.

// Implicit — object before the dot obj.foo(); // this = obj // Explicit — forced with call/apply/bind foo.call(obj); // this = obj // Arrow — inherits from surrounding scope const obj = { name: "React", getName: () => this.name // this is NOT obj! };
Say in Interview

"In modern React with hooks, this is less of an issue since we use function components. But understanding it is still important for: event handlers in class components (legacy code), understanding how bind was used in older React, and debugging callback patterns. Arrow functions inherit this lexically, which is why they work so well in React components."

5
What are Promises? Explain Promise.all, race, any, allSettled. Intermediate P1
Quick Answer

Think of a Promise like a coffee receipt: Pending (waiting), Fulfilled (☕ coffee ready), Rejected (😢 machine broke). You use .then() to handle success, .catch() for errors.

Promise helpers: all = wait for ALL (fails fast on any rejection). allSettled = wait for all (never fails, get all results). race = first to finish wins (resolve or reject). any = first success wins (ignores rejections).

// Promise.all — fail-fast const [users, posts] = await Promise.all([ fetch("/api/users"), fetch("/api/posts") ]); // Promise.allSettled — get all results even if some fail const results = await Promise.allSettled([ fetch("/api/users"), fetch("/api/might-fail") ]); results.forEach(r => { if (r.status === "fulfilled") console.log(r.value); else console.log(r.reason); });
Say in Interview

"I use Promise.all when I need multiple independent API calls in parallel — like fetching user data and posts at the same time. Promise.allSettled is great when some calls might fail but I still want the successful ones. Promise.race is useful for timeouts. In React, these are essential in useEffect data fetching patterns."

6
How does async/await work? Error handling compared to .then()? Intermediate P1
Quick Answer

Think of async/await as "syntactic sugar" — it's just a cleaner way to write Promises. An async function always returns a Promise. await pauses the function until the Promise resolves (but doesn't block the main thread).

Behind the scenes, await is just .then() with better formatting. Use try/catch for error handling instead of .catch() chains.

// Async/await — cleaner to read async function fetchUser(id) { try { const res = await fetch(`/api/users/${id}`); if (!res.ok) throw new Error(`HTTP ${res.status}`); return await res.json(); } catch (err) { console.error("Failed:", err); return null; } }
Say in Interview

"I always use async/await over Promise chains — it's more readable, especially when you have sequential steps. One trap: await inside a forEach doesn't work as expected because it doesn't respect promises. Use for...of for sequential or Promise.all with map for parallel execution."

7
What is hoisting? How does it differ for var, let, const, and function declarations? Intermediate P1
Quick Answer

Memory trick: "Declarations move up, assignments stay." During compilation, JavaScript moves all declarations to the top of their scope.

  • Function declarations — fully hoisted (can call before writing!)
  • var — hoisted and initialized as undefined
  • let/const — hoisted but NOT initialized (Temporal Dead Zone)
  • Arrow/expression functions — follow the hoisting of their variable
sayHi(); // ✅ "Hi" — fully hoisted function sayHi() { console.log("Hi"); } console.log(x); // undefined var x = 5; console.log(y); // ❌ ReferenceError — TDZ! let y = 5;
8
What is prototypal inheritance and the prototype chain? Advanced P1
Quick Answer

Think of it as a "fallback chain." If an object doesn't have a property, JavaScript checks its prototype (parent). If not there, it checks the prototype's prototype, and so on until it reaches null.

const animal = { eats: true }; const rabbit = Object.create(animal); rabbit.jumps = true; rabbit.jumps; // true (own property) rabbit.eats; // true (from prototype) // Chain: rabbit → animal → Object.prototype → null
Say in Interview

"JavaScript doesn't use classical inheritance like Java. It uses prototypal inheritance — objects inherit from other objects. Classes in ES6 are just syntactic sugar over prototypes. In React class components (legacy), understanding the prototype chain helps when extending base components."

map() transforms every element (same length). filter() picks elements that pass a test (shorter). reduce() combines all elements into one value (anything!). None of them mutate the original array.

10
Deep clone an object — limitations of JSON.parse/stringify? Intermediate P1
Quick Answer

JSON.parse(JSON.stringify(obj)) is the quick-and-dirty way to deep clone, but it drops: functions, undefined, Symbols, BigInt, circular references, and converts Date to string, Map/Set to {}.

Use structuredClone() (Node 17+/modern browsers) for proper deep cloning. It handles all of these.

// Limited — drops functions, Dates, Maps, etc. const clone1 = JSON.parse(JSON.stringify(original)); // Modern — handles everything const clone2 = structuredClone(original);
11
Event delegation, bubbling, and capturing? Intermediate P1
Quick Answer

Think of it as a ripple in water: Capturing = ripple spreading from the edge to the center (target). Bubbling = ripple bouncing back from center to the edge.

Event delegation = instead of adding a listener to each child, add ONE listener to the parent and check event.target. Saves memory and works for dynamically added elements.

// Event delegation — one listener instead of many document.querySelector("ul").addEventListener("click", e => { if (e.target.tagName === "LI") { console.log("Clicked:", e.target.textContent); } });
12
Arrow functions vs regular functions — this, arguments, constructors? Intermediate P1
Quick Answer

Arrow functions are "light" functions:

  • ❌ No own this — inherits from surrounding scope
  • ❌ No arguments object — use ...args
  • ❌ Can't be used as constructors with new
  • ❌ Can't be generators (no yield)
  • ✅ Implicit return: const add = (a,b) => a + b

In React, arrow functions are used everywhere — they avoid this confusion in class components and work perfectly as callbacks.

13
== vs === — when to use each in React? Beginner P1
Quick Answer

Memory trick: "Double = loose, Triple = strict." == converts types before comparing (coercion). === compares both value AND type.

Always use === in React. The only exception: value == null catches both null and undefined.

5 == "5" // true 5 === "5" // false 0 == false // true (bug waiting to happen!) 0 === false // false null == undefined // true null === undefined // false

Spread operator vs rest operator — React examples? Intermediate P1
Quick Answer

Memory trick: Spread = "SPread out" (unpacks). Rest = "collect the REST" (packs).

Both use the ... syntax but do opposite things.

// Spread — unpacks (used EVERYWHERE in React) const newObj = { ...oldObj, updatedField: value }; // State update const newArr = [...oldArr, newItem]; // Array update // Pass props // Rest — collects remaining function Component({ title, ...restProps }) { // restProps has everything except title return
{title}
; }
15
Implement debounce and throttle — when to use each in React? Advanced P1
Quick Answer

Debounce = wait for silence before running. Like search — don't search until the user stops typing. Throttle = limit rate. Like scrolling — run at most once every X milliseconds.

// useDebounce hook function useDebounce(value, delay = 300) { const [debounced, setDebounced] = useState(value); useEffect(() => { const timer = setTimeout(() => setDebounced(value), delay); return () => clearTimeout(timer); }, [value, delay]); return debounced; } // Usage const debouncedSearch = useDebounce(searchTerm, 300); useEffect(() => { if (debouncedSearch) fetchResults(debouncedSearch); }, [debouncedSearch]);
16
Nullish coalescing (??) vs OR (||) — React examples? Intermediate P1
Quick Answer

Key difference: ?? only replaces null/undefined. || replaces ALL falsy values (false, 0, '', null, undefined, NaN).

// React — use ?? when 0 or '' are valid values const count = props.count ?? 0; // ✅ 0 is a valid count const name = props.name ?? "Guest"; // ✅ '' is valid name // ❌ Wrong — || would replace 0 with default! const count2 = props.count || 0; // Replaces 0 with 0 (works but wrong logic)
17
Object.freeze() vs seal() vs preventExtensions()? Advanced P1
Quick Answer

Think of it as different levels of lockdown:

preventExtensions — can't add new properties (but can delete/modify). seal — can't add/delete (but can modify existing). freeze — can't add/delete/modify (read-only). All are shallow — nested objects can still change.

React use: Redux uses Object.freeze in development to catch accidental state mutations.

18
Map/WeakMap and Set/WeakSet differences? Advanced P1
Quick Answer

Weak versions allow garbage collection. WeakMap keys must be objects — when the object is no longer referenced elsewhere, it gets cleaned up. WeakMap has no .size and can't be iterated. Useful for attaching data to DOM elements without memory leaks.

React use: Map for lookup tables, Set for tracking selected items in lists.

19
What is NaN and how do you check for it? Intermediate P1
Quick Answer

Fun fact: NaN is the only value in JavaScript that is NOT equal to itself (NaN !== NaN is true). Use Number.isNaN() (NOT the global isNaN() which coerces values to numbers first).

isNaN("hello"); // true — coerces "hello" → NaN Number.isNaN("hello"); // false — correct, "hello" is NOT NaN Number.isNaN(NaN); // true typeof NaN; // "number" 🤯
20
call(), apply(), bind() — implement bind polyfill? Advanced P1
Quick Answer

Memory trick: call = "one by one", apply = "as array", bind = "make a copy for later."

// call — invoke with this + individual args greet.call(user, "Hello", "!"); // apply — invoke with this + args array greet.apply(user, ["Hello", "!"]); // bind — returns NEW function with this permanently set const boundGreet = greet.bind(user); boundGreet("Hello", "!"); // bind polyfill Function.prototype.myBind = function(context, ...args) { const fn = this; return function(...innerArgs) { return fn.apply(context, [...args, ...innerArgs]); }; };

P2 — Commonly Asked Questions

Questions often asked in technical screens and mid-level Frontend interviews.

21
Primitive vs reference types — why does React need immutability?BeginnerP2
Quick Answer

Think of primitives as Post-it notes (copied), references as house keys (shared). 7 primitives: string, number, boolean, null, undefined, symbol, bigint — stored by value. Objects/arrays/functions — stored by reference.

React context: When you mutate an object in state, React doesn't know it changed because the reference is the same. You must create a NEW reference to trigger re-render:

// ❌ Wrong — mutating, no re-render setState(prev => { prev.name = "John"; return prev; }); // ✅ Correct — new reference, triggers re-render setState(prev => ({ ...prev, name: "John" }));
22
Truthy and falsy values — conditional rendering pitfalls?BeginnerP2
Quick Answer

Only 6 falsy values: false, 0, '', null, undefined, NaN. Everything else is truthy — including [], {}, 'false'.

React pitfall: {items.length && } renders 0 when empty! Use {items.length > 0 && } or ternary.

// ❌ Renders "0" when items is empty! {items.length && } // ✅ Correct {items.length > 0 && } {items.length ? : null}
23
null vs undefined — when does each occur?BeginnerP2
Quick Answer

undefined = JavaScript's default \"I don't know what this is.\" null = you explicitly set \"this is empty.\" Check both: value == null catches both.

let a; // undefined const b = null; // null const obj = { x: 1 }; console.log(obj.y); // undefined (property doesn't exist) typeof null; // "object" — famous JS bug!
24
splice() vs slice() — why be careful in React state management?BeginnerP2
Quick Answer

Memory trick: slice = "sLice" (L = Less destructive — doesn't mutate). splice = "sPlice" (P = Permanent change — mutates).

slice() returns a new array (safe for React). splice() changes the original array (dangerous for React state!). NEVER use splice() on state arrays.

25
JSON.stringify and JSON.parse — React API/data uses?BeginnerP2
Quick Answer

stringify converts object → JSON string. parse converts JSON string → object. Used for: API responses, localStorage persistence, deep cloning (limited), debugging.

// localStorage with React useEffect(() => { localStorage.setItem("user", JSON.stringify(user)); }, [user]); const saved = JSON.parse(localStorage.getItem("user") || "null");
26
try/catch/finally — React error boundaries vs async errors?BeginnerP2
Quick Answer

try wraps risky code. catch handles errors. finally always runs (cleanup). In React, try/catch works in event handlers and useEffect but NOT during rendering — use Error Boundaries for rendering errors.

useEffect(() => { async function load() { try { const res = await fetch("/api/data"); setData(await res.json()); } catch (err) { setError(err.message); } finally { setLoading(false); } } load(); }, []);

What is an IIFE — why would you use one?IntermediateP2
Quick Answer

"Self-destructing function" — runs immediately and disappears. Creates private scope before ES6 modules existed. In React, useful for initializing state with computation or running async code at module level.

// Lazy initial state in React const [data] = useState(() => { // This IIFE runs only once on mount const cached = localStorage.getItem("data"); return cached ? JSON.parse(cached) : null; });
28
Currying — how is it useful in React?AdvancedP2
Quick Answer

Currying = function that takes arguments one at a time instead of all at once. f(a, b, c) becomes f(a)(b)(c). In React: HOCs (Higher-Order Components), event handlers with extra params, Redux middleware pattern (store => next => action => {}).

Remove duplicates from an array — compare approaches?IntermediateP2
Quick Answer

For simple values: [...new Set(arr)] — one line, O(n). For objects: Use Map with a unique key or filter + findIndex.

const simple = [...new Set([1, 2, 2, 3])]; // [1, 2, 3] const unique = arr.filter( (item, i, self) => self.findIndex(t => t.id === item.id) === i );

Memoization — implement and React equivalents?AdvancedP2
Quick Answer

"Write it down so you don't calculate again." Memoization caches function results by arguments. In React: React.memo (component memoization), useMemo (value memoization), useCallback (function memoization).

// React memoization const MemoizedComponent = React.memo(MyComponent); const expensiveResult = useMemo(() => { return data.filter(d => d.active).map(transform); }, [data]); const memoizedCallback = useCallback(() => { doSomething(id); }, [id]);

map() returns a new array (use for React rendering!). forEach() returns undefined (use for side effects). In React, ALWAYS use map() for rendering lists.

34
localStorage vs sessionStorage vs cookies?IntermediateP2
Quick Answer

localStorage: persists across tabs until cleared. sessionStorage: per-tab, cleared on close. Cookies: sent with HTTP requests, max 4KB, can expire. React use: localStorage for theme, auth (cautiously), cached data.

35
typeof null, typeof [], typeof NaN?BeginnerP2
Quick Answer

typeof null → "object" (bug). typeof [] → "object" (use Array.isArray()). typeof NaN → "number" (use Number.isNaN()).

36
Pure vs impure function — why matter for React?IntermediateP2
Quick Answer

Pure functions: same input → same output, no side effects. React components should be pure — given same props, render same output. Side effects go in useEffect. Redux reducers must be pure. React's performance optimizations rely on purity.

37
Flatten a nested array — flat() and recursive?IntermediateP2
Quick Answer

arr.flat(depth) or arr.flat(Infinity). Recursive: use reduce + concat. Useful for normalizing nested API data in React.

38
Generators — when would you use them?AdvancedP2
Quick Answer

"Functions that can pause and resume." function* with yield. Less common in React but useful for: infinite loading, generating unique IDs, Redux Saga (legacy).

39
Short-circuit evaluation — React conditional rendering?BeginnerP2
Quick Answer

&& returns first falsy or last truthy. || returns first truthy or last falsy. React uses && for conditional rendering: {isLoggedIn && }. BUT watch out for falsy values!

40
for...in vs for...of?BeginnerP2
Quick Answer

for...in loops over KEYS (use for objects). for...of loops over VALUES (use for arrays, strings, Maps). In React, prefer .map() for rendering or for...of with await for sequential async.

41
Type coercion — implicit vs explicit examples?IntermediateP2
Quick Answer

Implicit: '5' - 3 = 2, '5' + 3 = '53', !0 = true. Explicit: Number('5'), String(5), Boolean(0). Be careful with form values — they're always strings!

42
Array.isArray() vs instanceof Array?IntermediateP2
Quick Answer

Use Array.isArray() — it's reliable across frames/contexts. instanceof Array can fail if the array comes from a different window/iframe.

43
Check if object has a property — own vs inherited?IntermediateP2
Quick Answer

Object.hasOwn(obj, 'key') or obj.hasOwnProperty('key') — own properties only. 'key' in obj — own + inherited. Don't use obj.key !== undefined — the key might exist with undefined value.

Optional chaining and nullish coalescing — React safety patterns?IntermediateP2
Quick Answer

Optional chaining (?.): "Only go deeper if it exists." Prevents "Cannot read property of undefined." Nullish coalescing (??): default only for null/undefined.

P3 — Occasionally Asked (Valuable Context)

Less common but useful for senior roles and deep-dive interview rounds.

46
What is strict mode ("use strict") — what does it prevent?IntermediateP3
Quick Answer

Like having a strict teacher — catches mistakes that a lenient teacher ignores. Prevents: undeclared variables, deleting non-deletable properties, duplicate params, this being global, with statement. React uses strict mode internally in dev.

47
What are Symbols — React/React internals use case?AdvancedP3
Quick Answer

Symbols are unique, immutable property keys. React uses Symbol.for('react.element') internally. Useful for creating truly unique keys and avoiding property name collisions in libraries.

48
Proxy API — Immer use case in React state management?ExpertP3
Quick Answer

Proxy intercepts object operations. Immer uses Proxy extensively — when you write mutable-looking code inside produce(), Immer intercepts all mutations and creates immutable snapshots. This is how you can write "mutating" state updates that are actually immutable.

// Immer with Proxy under the hood const nextState = produce(state, draft => { draft.user.name = "John"; // Looks like mutation, but creates new state! draft.items.push(newItem); });
49
requestAnimationFrame — React animations use case?AdvancedP3
Quick Answer

Schedules code before the next browser repaint (~60fps). Pauses when tab is inactive (saves battery). Use for: animations, canvas rendering, scroll-based animations. Can be used in custom React animation hooks.

50
Web Workers — React heavy computation use case?ExpertP3
Quick Answer

Web Workers run JS in a background thread with their own event loop. Can't access DOM. Use for: heavy data processing (CSV parsing, image manipulation), large list filtering, WebAssembly. Communication via postMessage.

51
setTimeout vs setInterval — React pitfalls?BeginnerP3
Quick Answer

setTimeout runs once after delay. setInterval runs repeatedly. In React, always clean up timers in useEffect cleanup to prevent memory leaks and stale closures.

52
Deep vs shallow comparison in React?IntermediateP3
Quick Answer

React uses shallow comparison by default (===) for props. React.memo does shallow comparison. useEffect deps use Object.is(). Deep comparison is expensive — avoid it. Use immutable updates so shallow comparison works automatically.

53
Object.create(null) — why create an object with no prototype?AdvancedP3
Quick Answer

Object.create(null) creates an object with NO prototype chain — it doesn't inherit any properties. Useful for dictionary/safe-map patterns where you don't want inherited keys like toString or hasOwnProperty interfering.

54
Reflect API — how does it relate to Proxy?ExpertP3
Quick Answer

Reflect provides methods matching every Proxy trap. Use inside Proxy traps to forward default behavior. return Reflect.get(target, prop, receiver). Used by Immer internally.

55
React Synthetic Events — how does React wrap native events?AdvancedP3
Quick Answer

React wraps native browser events in Synthetic Events for cross-browser compatibility. In React 17+, events are attached to the root React container instead of the document. In React 16 and earlier, Synthetic Events were pooled for performance (event properties cleared after callback) — this is removed in React 18.
