P1 — Frequently Asked Questions

1
Design the architecture of a large-scale React application (10M+ users).P1
Basic Answer

Key considerations: Monorepo (Turborepo/Nx) for shared packages, Micro-frontends or modular monolith, Feature-based folder structure, Shared UI component library (Storybook). Performance: code splitting, CDN, ISR/SSR (Next.js), lazy loading. State management: Zustand (client) + React Query (server). Testing: RTL + Playwright. CI/CD: GitHub Actions + Vercel deployment. Monitoring: Sentry, Datadog RUM.

2
Explain micro-frontends. When should you use them? Trade-offs?P1
Basic Answer

Micro-frontends decompose large frontend apps into smaller, independent apps owned by separate teams. Each micro-frontend can use its own framework, have its own CI/CD, and be deployed independently. Integration approaches: iframes, Web Components, Module Federation, NPM packages, or edge-side includes. Trade-offs: increased complexity, shared dependencies, consistent UX challenges, duplicated code. Start with a modular monolith first — only split when the monolith becomes unsustainable.

3
How do you optimize for Core Web Vitals (LCP, CLS, INP)?P1
Basic Answer

LCP (Largest Contentful Paint): optimize images (next/image, WebP, responsive sizes), preload hero images, minimize render-blocking resources, use CDN. CLS (Cumulative Layout Shift): set width/height on images (aspect-ratio), reserve space for ads/dynamic content, use font-display: optional, avoid inserting content above existing content. INP (Interaction to Next Paint): avoid long tasks, break up heavy computation, use useTransition for non-urgent updates, optimize event handlers.

4
How would you design a real-time collaborative application (like Google Docs)?P1
Basic Answer

Architecture: CRDT (Conflict-Free Replicated Data Types) or OT (Operational Transform) for conflict resolution. WebSocket for real-time communication. Debounced local updates. Cursor presence using broadcast channel. State snapshot + incremental ops. React: useSyncExternalStore for external state, React 19 useOptimistic for optimistic updates. Libraries: Yjs (CRDT), ShareDB (OT). Handle: offline editing, reconnection, cursor synchronization.

5
How do you handle frontend security? (XSS, CSRF, CSP, Auth)P1
Basic Answer

XSS: React escapes JSX by default (but not dangerouslySetInnerHTML), sanitize user input with DOMPurify, Content Security Policy headers. CSRF: SameSite cookies, CSRF tokens. CSP: restrict script sources, report violations. Auth: secure HTTP-only cookies, token rotation, proper OAuth2 flows. Never store sensitive data in localStorage. Use next/headers for auth in Next.js Server Components.

6
How would you design a scalable component library?P1
Basic Answer

Structure: atomic design (atoms, molecules, organisms). Package: publish to NPM (private registry). Build: TypeScript + Rollup/Vite library mode, CSS Modules or Tailwind CSS. Documentation: Storybook with stories for all states, Chromatic for visual regression. Testing: RTL + jest-axe. Versioning: SemVer + Changesets. Theming: CSS custom properties, design tokens. Accessibility: WCAG 2.1 AA, keyboard navigation, ARIA attributes.

7
How do you approach accessibility (a11y) at scale?P1
Basic Answer

Semantic HTML, proper ARIA roles/states, keyboard navigation, focus management, color contrast (WCAG 2.1 AA/AAA), screen reader announcements, alternative text for images. Testing: jest-axe (unit), @axe-core/playwright (E2E), manual testing with screen readers. Build a11y into component library. Use automated checks in CI. Design handoff includes accessibility annotations.

8
How do you implement an effective caching strategy for a React app?P1
Basic Answer

Browser cache: Service Worker caching (workbox), Cache-Control headers for static assets (immutable). CDN cache: CloudFront/Cloudflare for static files. React Query: stale-while-revalidate, background refetching. Next.js: Data Cache, Full Route Cache, Router Cache. localStorage/indexedDB for offline-first. SWR pattern for API responses. Proper cache invalidation with revalidatePath/revalidateTag.

9
How would you design an infinite scroll feed with 100k+ items?P1
Basic Answer

Virtualization (react-virtuoso or tanstack-virtual) — only render visible items + buffer. Intersection Observer for triggering more loads. React Query infinite queries for pagination. Cursor-based pagination (not offset — performant with large datasets). Skeleton loading for new pages. Memory management: window the data (remove items far from viewport). Server-side: efficient cursor-based SQL queries.

10
What is the role of CDN in frontend architecture?P1
Basic Answer

CDN (Content Delivery Network) caches static assets at edge locations worldwide. Benefits: faster load times (reduced latency), reduced server load, DDoS protection, SSL termination. Strategies: cache static JS/CSS/images aggressively (immutable), use CDN for fonts, serve dynamic content via CDN with TTLs. CloudFront (AWS), Cloudflare, Fastly, Vercel Edge Network. Proper cache headers are essential.

11
How do you handle error tracking and monitoring in production React apps?P1
Basic Answer

Sentry: error tracking, source maps, performance monitoring. Datadog RUM: real user monitoring. New Relic Browser. LogRocket: session replay + errors. Key: structured error logging, source maps for debugging production errors, error grouping, alerting on error thresholds. React Error Boundaries to catch rendering errors. APM for monitoring Core Web Vitals.

12
How do you handle SEO for a React SPA vs Next.js SSR?P1
Basic Answer

SPA: Prerender (react-snap, prerender-spa-plugin), SSR via rendering service (Rendertron), or use Next.js. Next.js: SSR/SSG for full SEO, metadata API, sitemap generation, canonical URLs, structured data (JSON-LD), Open Graph images. Meta tags: title, description, OG tags. Server Components render SEO-relevant content. Dynamic OG images with @vercel/og. Google crawler handles JS (but SSR is better).

P2 — Commonly Asked Questions

13
How do you implement PWA features in a React app?P2
Basic Answer

Service Worker (Workbox for easier setup), manifest.json, offline fallback page, cache-first strategy for static assets, network-first for API, background sync for offline actions, push notifications. Next.js: next-pwa or @serwist/next. PWA checklist: HTTPS, responsive, works offline, installable (manifest + SW).

14
How do you handle internationalization (i18n) at scale?P2
Basic Answer

Translation management: next-intl, react-i18next, formatjs. Translation files per locale, CI/CD integration for translation updates. Right-to-left (RTL) support. Date/number formatting with Intl. SEO: hreflang tags, localized URLs (/en/page, /fr/page). Next.js: middleware-based locale detection, static generation per locale. Context-aware translations (pluralization, gender).

15
How do you design a form system for complex enterprise applications?P2
Basic Answer

Schema-driven forms (react-jsonschema-form, Formik/Yup, React Hook Form + Zod). Dynamic field rendering based on schema. Conditional fields (show/hide based on values). Multi-step wizard with progress persistence. Validation: client-side (immediate) + server-side (on submit). Auto-save drafts. Accessibility: proper labels, error announcements, focus management. React Hook Form for performance.

16
How do you handle state synchronization between browser tabs?P2
Basic Answer

BroadcastChannel API for messaging between tabs. storage event for localStorage changes (catches changes from other tabs). SharedWorker for persistent cross-tab communication. Zustand persist middleware with cross-tab sync. Use cases: auth state (logout from one tab, logged out from all), theme changes, user preferences. Be careful with race conditions during sync.

17
How would you design a notification system?P2
Basic Answer

Component: toast/notification component with position, variant (success/error/info/warning), auto-dismiss, stack management, accessibility (announce via aria-live). State: notification queue with max visible count. Context for global access. Animation: mount/unmount animations (Framer Motion AnimatePresence). WebSocket for real-time notifications from server. Prioritize critical notifications.

18
How do you design for offline-first?P2
Basic Answer

Service Worker for caching pages and assets. IndexedDB for storing application data (Dexie.js wrapper). Queue offline mutations for sync when online. State management: persist state to IndexedDB. Detect online/offline status (navigator.onLine, online/offline events). Show UI indicators for offline mode. Conflict resolution when syncing. React Query: networkMode: 'offlineFirst'.

19
How do you design for different device types (responsive + adaptive)?P2
Basic Answer

Mobile-first CSS with breakpoints. Touch vs mouse interactions (hover, pointer events). Responsive images (srcSet, sizes). Adaptive components (render different UI for mobile vs desktop). Tailwind responsive prefixes. Device detection (useMediaQuery hook). Touch events vs click events. Performance considerations for mobile (reduce JS, optimize images).

20
How do you handle log management and debugging in production?P2
Basic Answer

Structured logging (console.log → custom logger with levels). Log shipping to ELK/Datadog. Source maps for error stack traces. Session replay (LogRocket, FullStory). RUM (Real User Monitoring) for performance. Feature flags for gradual rollouts. A/B testing experimentation platform. Environment-specific logging levels. User behavior analytics.

P3 — Occasionally Asked (Valuable Context)

21
What is Edge computing and how does it apply to frontend?P3
Basic Answer

Edge computing runs code close to the user geographically. Vercel Edge Functions, Cloudflare Workers, AWS Lambda@Edge. Use cases: A/B testing, personalization, redirects, auth checks, API response transformation. Edge can improve performance by reducing round-trip time. Trade-offs: limited runtime (no Node.js APIs, cold starts), smaller bundle size limit.

22
How would you design a search feature with autocomplete?P3
Basic Answer

Debounced input (300ms). API request with query parameter. Loading state, empty state, error state, results. Keyboard navigation (arrow keys, enter to select, escape to close). Highlight matching text in results. Click outside to close. Accessibility: combobox role, aria-activedescendant. Caching recent searches. Analytics for search terms.

23
How do you implement data visualization dashboards efficiently?P3
Basic Answer

Chart libraries: Recharts (React-native), D3 (low-level, flexible), visx (Airbnb), Nivo, Tremor (Tailwind charts). Performance: data aggregation on server, canvas/SVG-based rendering, avoid re-rendering unchanged charts (React.memo). Lazy load charts below the fold. Web Workers for heavy data transformations. Responsive charts. Accessibility: provide data tables as alternatives.

24
How do you handle rate limiting and API throttling from the frontend?P3
Basic Answer

Retry logic with exponential backoff (don't retry on 429 immediately). Queue failed requests for retry. Show user-facing throttling indicators. Debounce/throttle user-triggered API calls. Circuit breaker pattern (stop calling API after certain failure threshold). Cache responses to avoid duplicate requests. React Query handles retry+backoff automatically.

25
How would you design a drag-and-drop interface?P3
Basic Answer

HTML5 Drag and Drop API (native, but limited). React DnD (flexible, higher learning curve). @dnd-kit (modern, accessible, recommended — used by Trello clone). Features: sorting, drag handle, drop zones, axis constraints, animations, multi-select drag, scroll on drag, accessibility (keyboard reordering). Touch support for mobile.
