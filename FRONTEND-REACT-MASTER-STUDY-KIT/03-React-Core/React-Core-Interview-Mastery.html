P1 — Frequently Asked Questions

You MUST know these for any React interview.

1
What is React? Explain the Virtual DOM and how it works.IntermediateP1
Basic Answer

React is a declarative, component-based UI library. The Virtual DOM is a lightweight JavaScript representation of the real DOM. When state changes, React creates a new Virtual DOM tree, diffs it against the previous one (reconciliation), calculates minimal updates, and applies them to the real DOM (commit phase). This process is called "reconciliation" and is powered by the Fiber architecture.

Real Interview Answer

"React's Virtual DOM is a lightweight copy of the real DOM in memory. When state changes, React computes the difference (diffing) between the new and old Virtual DOM trees using the Fiber reconciler, then applies only the minimal updates to the actual DOM. This makes React efficient because direct DOM manipulation is slow. The key insight is that React batches these updates and uses a sophisticated reconciliation algorithm with keys for efficient list updates."

2
What is JSX? How does it work under the hood?IntermediateP1
Basic Answer

JSX is a syntax extension that looks like HTML but compiles to React.createElement() calls. Babel transforms JSX into React.createElement(type, props, ...children) which returns a React element (a plain object with type, props, and key). JSX makes React code more readable. React 17+ introduced the new JSX transform that auto-imports from react/jsx-runtime, so you don't need import React at the top of every file.

3
Explain the difference between functional components and class components.IntermediateP1
Basic Answer

Functional components are plain functions that accept props and return JSX. Class components extend React.Component and have lifecycle methods. With hooks (React 16.8+), functional components can do everything class components can — state (useState), side effects (useEffect), context (useContext), and more. Functional components are simpler, have less boilerplate, and the React team recommends them for all new code.

4
Explain the React component lifecycle and how hooks replace lifecycle methods.AdvancedP1
Basic Answer

Lifecycle phases: Mounting, Updating, Unmounting. Class component methods: componentDidMount, componentDidUpdate, componentWillUnmount. With hooks: useEffect(() => {}, []) = componentDidMount. useEffect(() => {}) = componentDidUpdate (runs on every render). useEffect(() => { return () => {} }, []) cleanup = componentWillUnmount. Multiple useEffect calls separate concerns better than putting everything in one lifecycle method.

// Class Component componentDidMount() { fetchData(); } componentDidUpdate(prevProps) { if (prevProps.id !== this.props.id) fetchData(); } componentWillUnmount() { cleanup(); } // Functional Component with Hooks useEffect(() => { fetchData(); }, [id]); // Runs on mount + when id changes useEffect(() => { return () => cleanup(); // Cleanup on unmount }, []);
5
What is useState? How does it work? Explain the closure trap.IntermediateP1
Basic Answer

useState returns a state variable and a setter function. The setter triggers a re-render with the new state. State is preserved between renders through React's Fiber nodes (not closures). Each render has its own snapshot of state — closures capture the state at that render's time. The "stale closure" trap happens when async code (setTimeout, Promise) inside an effect captures an old state value. Use the functional updater form setCount(prev => prev + 1) to avoid this.

6
What is useEffect? Explain the dependency array and cleanup function.IntermediateP1
Basic Answer

useEffect runs side effects after render. Dependency array controls when it re-runs: [] = run once on mount, [dep] = run when dep changes, no array = run after every render. Cleanup function runs before re-run and on unmount. Common mistakes: missing dependencies (use lint rules!), including unnecessary dependencies, using objects/arrays as dependencies (new reference every render).

7
What is useRef? When would you use it?IntermediateP1
Basic Answer

useRef returns a mutable object ({ current: initialValue }) that persists across renders. Changing .current does NOT trigger re-renders. Uses: accessing DOM elements, storing mutable values that shouldn't cause re-renders (timer IDs, previous state values), and holding instance-like variables in functional components.

8
What is useReducer? When should you use it instead of useState?IntermediateP1
Basic Answer

useReducer is for complex state logic — multiple sub-values, transitions between states, or when state depends on previous state. Pattern: useReducer(reducer, initialState) where reducer takes (state, action) => newState. Use when state logic is complex (forms with many fields), when state transitions depend on each other, or when you want to centralize state logic for easier testing.

9
Explain controlled vs uncontrolled components in forms.IntermediateP1
Basic Answer

Controlled: React state manages the form value (value={state} onChange={handler}). The React state is the "single source of truth." Uncontrolled: the DOM manages its own state — use useRef to read values when needed. Prefer controlled components for validation, instant feedback, and complex forms. Use uncontrolled for simple forms, file inputs, or integrating with non-React code.

10
What are keys in React lists? Why is the index an anti-pattern?IntermediateP1
Basic Answer

Keys help React identify which items changed, were added, or removed. Use stable, unique IDs from your data. Using array index as key is an anti-pattern because: when items are reordered, added, or removed, React incorrectly reuses component instances, causing state bugs and performance issues. Only use index as a last resort for static lists that won't change.

11
What is Context API? When should you use it?IntermediateP1
Basic Answer

Context provides a way to pass data through the component tree without prop drilling. createContext → Context.Provider → useContext(Context). Use for: themes, user auth, locale/preferences, and UI state (sidebars, modals). DON'T use for: frequent updates (causes re-renders of all consumers), complex state logic (use a state management library instead), or as a replacement for prop drilling in small component trees.

12
Explain React's reconciliation algorithm (diffing).ExpertP1
Basic Answer

Reconciliation is the process of comparing two Virtual DOM trees. React uses two assumptions (heuristics): (1) Different element types produce different trees — React rebuilds the entire subtree. (2) Stable keys indicate stable elements across renders. The diffing algorithm: compare root elements, if different → unmount old tree, mount new tree. If same type → update props, recurse on children matched by keys.

13
What are Fragments? Why use React.Fragment or <>?BeginnerP1
Basic Answer

Fragments let you return multiple elements without adding extra DOM nodes. Use <>... or . keyed fragments () need explicit Fragment syntax. Benefits: avoids unnecessary wrapper divs, keeps DOM cleaner, improves accessibility (no extra nesting for screen readers).

14
What is conditional rendering? Compare approaches.BeginnerP1
Basic Answer

Conditional rendering approaches: ternary ({condition ? : }), logical AND ({condition && }), if/else outside JSX, IIFE, ternary with null ({condition ? : null}). Pitfalls: {0 && } renders 0 — use explicit conditions like {count > 0 && }.

15
Explain React's event handling system (Synthetic Events).AdvancedP1
Basic Answer

React wraps native events with SyntheticEvent objects for cross-browser consistency. In React 16 and earlier, events were pooled and attached to the document. React 17+ attaches events to the root container (not document) and removed event pooling. Modern React doesn't use event pooling — e.persist() is unnecessary. Event handlers receive SyntheticEvent with preventDefault(), stopPropagation(), etc.

16
What are Portals? When would you use them?AdvancedP1
Basic Answer

Portals (createPortal(children, domNode)) render children into a different DOM subtree outside the parent component. Use cases: modals, tooltips, dropdowns, toasts — any overlay that needs to escape parent's overflow/stacking context. Portals preserve React context and event bubbling through the React tree, not the DOM tree.

17
What is StrictMode? What does it do?IntermediateP1
Basic Answer

StrictMode is a development-only wrapper that helps catch bugs. It: double-invokes render functions and effects (to detect impure logic), checks for deprecated APIs, and warns about unsafe lifecycle methods. The double-invocation is important — it helps find side effects in rendering (which should be pure). Only affects development mode.

18
What is Lifting State Up? Why is it important?IntermediateP1
Basic Answer

Lifting state up means moving shared state to the closest common ancestor component so multiple children can share and sync state. This is React's fundamental data flow pattern — state flows down via props, and changes flow up via callbacks. When siblings need to communicate, lift state to their parent.

19
What is prop drilling? How do you solve it?IntermediateP1
Basic Answer

Prop drilling is passing props through multiple intermediate components that don't need them. Solutions: (1) Context API for global/shared data, (2) Component composition — pass components as children/props, (3) State management libraries (Zustand, Redux), (4) Lifting state too high vs keeping state local. Composition is often the cleanest solution.

20
How do you handle errors in React? Explain Error Boundaries.AdvancedP1
Basic Answer

Error Boundaries are class components that catch JavaScript errors in their child component tree during rendering, lifecycle methods, and constructors. They use componentDidCatch(error, info) and getDerivedStateFromError(error). They don't catch: event handlers (use try/catch), async code, server-side rendering, or errors in the error boundary itself. React 19 is introducing error boundaries for functional components.

21
Explain composition vs inheritance in React.IntermediateP1
Basic Answer

React strongly favors composition over inheritance. Composition patterns: Containment (children prop for generic containers), Specialization (function Wizard({ step, children }) — wizard frames), and Passing components as props (} body={}/>). In 7+ years of React, Facebook found no use cases for component inheritance hierarchies.

22
What is forwardRef and when would you use it?AdvancedP1
Basic Answer

forwardRef allows parent components to pass a ref through to a child's DOM element. Used for: reusable component libraries (input, button), focusing inputs from parent, measuring DOM nodes in children, and integrating with third-party DOM libraries.

P2 — Commonly Asked Questions

23
What are the rules of hooks? Why are they important?IntermediateP2
Basic Answer

Rules: (1) Only call hooks at the top level — not inside conditions, loops, or nested functions. (2) Only call hooks from React function components or custom hooks. The order of hooks must be consistent across renders because React relies on hook call order to match state between renders (linked list in Fiber).

24
What is useCallback and when should you use it?AdvancedP2
Basic Answer

useCallback(fn, deps) returns a memoized callback that only changes when deps change. Used to prevent unnecessary re-renders when passing callbacks to optimized child components (React.memo). Don't overuse — the overhead of memoization can exceed the cost of re-creation for simple functions.

25
What is useMemo and when should you use it?AdvancedP2
Basic Answer

useMemo memoizes computed values. Use for expensive calculations (data transformations, filtering large arrays, complex derivations). Don't use for trivial calculations — the comparison overhead outweighs benefits. Profile before optimizing.

26
What is React.memo? How is it different from useMemo?AdvancedP2
Basic Answer

React.memo is a HOC that memoizes the entire component — only re-renders when props change (shallow comparison). useMemo memoizes a value inside a component. React.memo is for preventing re-renders, useMemo is for preventing recalculations.

27
How does React batch state updates? Explain automatic batching.AdvancedP2
Basic Answer

React batches state updates that happen within the same synchronous context — event handlers, lifecycle methods. React 18 introduced automatic batching for all contexts (promises, setTimeout, native events) using a microtask-based mechanism. Multiple setState calls produce a single re-render, improving performance.

28
Explain how data flows in React (unidirectional flow).BeginnerP2
Basic Answer

React uses one-way (unidirectional) data flow. State is passed down the component tree via props. Child components communicate upward via callback functions passed as props. This makes data flow predictable and easier to debug compared to two-way binding.

29
What is the difference between Shadow DOM and Virtual DOM?AdvancedP2
Basic Answer

Virtual DOM: React's lightweight JS representation of the DOM for diffing and batching updates. Shadow DOM: browser API for encapsulating DOM and CSS in web components. They solve different problems — Virtual DOM for performance, Shadow DOM for encapsulation. React doesn't use Shadow DOM, but can render into Shadow DOM for web component integration.

30
How do you optimize performance in a React app?AdvancedP2
Basic Answer

Strategies: React.memo, useMemo/useCallback, virtualization (react-window), code splitting (lazy + Suspense), avoiding unnecessary re-renders, proper key usage, useTransition for non-urgent updates, avoiding context for frequently updating values, and profiling with React DevTools.

31
What is code splitting in React? How does lazy + Suspense work?IntermediateP2
Basic Answer

Code splitting breaks the bundle into smaller chunks loaded on demand. React.lazy(() => import('./Component')) enables dynamic imports. }> shows a fallback while loading. This reduces initial bundle size and improves load time. Combine with route-based splitting for the biggest impact.

32
Explain the different ways to style React components.BeginnerP2
Basic Answer

Approaches: CSS Modules (scoped styles), Styled Components (CSS-in-JS), Tailwind CSS (utility-first), inline styles (object-based), global CSS (traditional), and CSS-in-JS libraries (Emotion). React doesn't enforce any approach. Tailwind CSS is currently the most popular for new projects.

33
How do you handle forms in React? Compare libraries.IntermediateP2
Basic Answer

Manual: useState for each field, onSubmit handler. Libraries: React Hook Form (performant, minimal re-renders), Formik (popular, good validation), and Final Form. React Hook Form is currently most popular — uses refs and uncontrolled components for performance. For validation: Zod or Yup schemas.

34
Explain useImperativeHandle and when to use it.ExpertP2
Basic Answer

useImperativeHandle(ref, createHandle, deps) customizes the instance value exposed to parent refs. Used with forwardRef. Use cases: exposing focus/scroll methods, custom input components that need to integrate with parent form logic, and animation libraries that need direct access to component methods.

35
What are the different ways to pass data between components?IntermediateP2
Basic Answer

Parent → Child: props. Child → Parent: callback props. Siblings: lift state up. Distant: Context API. Complex: state management (Zustand, Redux). Server: React Query. URL: Next.js/Layout data. Event bus (not recommended — leads to spaghetti).

36
What is the children prop? Explain render props pattern.AdvancedP2
Basic Answer

The children prop is a special prop for nested JSX content. Render props is a pattern where a component receives a function as a prop (or children) that returns JSX, giving the parent control over rendering. Largely replaced by hooks and custom hooks, but still relevant for some component library patterns.

37
What is the difference between state and props?BeginnerP2
Basic Answer

Props are passed IN (immutable, from parent). State is internal (mutable, managed by the component). Props are like function arguments, state is like local variables. Changing props causes re-render (parent re-render), changing state causes re-render (setState call).

38
How do you create custom hooks? Share examples.AdvancedP2
Basic Answer

Custom hooks are functions that use React hooks and start with use. They encapsulate reusable stateful logic. Examples: useLocalStorage, useDebounce, useFetch, useMediaQuery, useIntersectionObserver, useAuth. Custom hooks are the primary way to share logic between components.

39
Explain the render cycle — what happens when state changes?AdvancedP2
Basic Answer

Phase 1: Trigger (state change, parent re-render, force update). Phase 2: Render (React calls component function, creates Virtual DOM tree). Phase 3: Reconciliation (diff old vs new Virtual DOM). Phase 4: Commit (apply DOM updates). Phase 5: useEffect (run side effects after paint). React 18 added "concurrent" features where rendering can be interrupted.

40
What are Higher-Order Components (HOCs)? Compare with hooks.AdvancedP2
Basic Answer

HOCs are functions that take a component and return an enhanced component: const Enhanced = withFeature(Wrapped). They wrap logic around components. Hooks are preferred over HOCs because they: avoid wrapper hell, are more composable, have no naming collisions, and are easier to type. Still used in some library APIs (connect in Redux, withStyles in MUI).

P3 — Occasionally Asked (Valuable Context)

41
What is React Fiber? How does it enable concurrent features?ExpertP3
Basic Answer

Fiber is the reimplementation of React's reconciliation algorithm (React 16+). It enables: incremental rendering (splitting work into chunks), ability to pause/abort work, prioritizing different updates, and reusing completed work. Each Fiber node represents a unit of work with state, side effects, and links to siblings/children. This architecture powers concurrent features like useTransition, Suspense, and automatic batching.

42
What is useDeferredValue and useTransition?ExpertP3
Basic Answer

useTransition — marks state update as low priority, keeps UI responsive. useDeferredValue — defers re-rendering of a value. Both are concurrent features for keeping UI responsive during expensive updates. Use for: search inputs with large result lists, tab switching, and filtering large datasets.

43
How does React 19 differ from React 18? What new features are coming?AdvancedP3
Basic Answer

React 19: use() hook for reading promises/context directly in render, Server Components (stable), useOptimistic for optimistic updates, useFormStatus for form state, new

actions, improved error handling, and document metadata support. React Canary (pre-release) includes these features.

44
What is flushSync and when would you need it?ExpertP3
Basic Answer

flushSync forces React to flush updates synchronously. Used when you need the DOM updated immediately — like when measuring DOM after a state change (getBoundingClientRect). Overuse breaks the benefits of async rendering. Only use as escape hatch.

45
How does React handle server-side rendering (SSR)?AdvancedP3
Basic Answer

React renders components to HTML strings on the server (renderToString or renderToPipeableStream). Client receives HTML (faster FCP), then React hydrates — attaches event handlers to the existing HTML. Next.js handles SSR elegantly with the App Router using Server Components and selective hydration.

46
What is hydration? What are common hydration issues?AdvancedP3
Basic Answer

Hydration is the process where React attaches event listeners and resumes component state on server-rendered HTML. Common issues: mismatched HTML (different on server vs client — fixed by using suppressHydrationWarning or ensuring deterministic output), hydration errors from dynamic content (dates, user-specific data), and performance (hydrating large component trees).

47
What is the key prop used for besides lists?AdvancedP3
Basic Answer

Keys can force component reset by changing the key — React treats it as a new component, unmounting the old and mounting a new one. Useful for: resetting state when data changes, re-mounting a component to restart animations, and forcing forms to reset. Pattern: — resets chat when user changes.

48
How would you implement an undo/redo feature in React?AdvancedP3
Basic Answer

Pattern: keep a history array of states and a pointer to current position. useReducer works well: undo → move pointer back, redo → move forward, apply → slice off future states and add new. Use a custom hook useHistory to encapsulate the logic. Be careful with large state objects — consider deep cloning strategies.

49
What is the difference between React.PureComponent and React.Component?IntermediateP3
Basic Answer

PureComponent implements shouldComponentUpdate with a shallow prop/state comparison. Component always re-renders when parent re-renders. In functional components, React.memo is the equivalent of PureComponent. Shallow comparison works when using immutable updates.

50
How does createElement work? What does a React element look like?ExpertP3
Basic Answer

React.createElement(type, props, ...children) returns a plain object: { $$typeof: Symbol(react.element), type, props, key, ref }. $$typeof is a Symbol that helps prevent XSS attacks (malicious JSON can't create valid elements). This is the fundamental building block — JSX is syntactic sugar for these calls.
