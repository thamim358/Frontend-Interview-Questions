P1 — Frequently Asked Questions

1
What is Next.js? What problems does it solve?IntermediateP1
Basic Answer

Next.js is a React framework for production. It solves: SSR/SSR for SEO and performance, file-based routing, API routes, image optimization, font optimization, and built-in CSS support. The App Router (Next.js 13+) adds Server Components, streaming, and layout-based routing. Vercel is the creator.

2
Explain App Router vs Pages Router — key differences and migration.AdvancedP1
Basic Answer

App Router (13+): Server Components by default, nested layouts, streaming, file-based routing with folders, loading.tsx, error.tsx, parallel routes, route groups. Pages Router: file-based, getServerSideProps/getStaticProps, _app.tsx, _document.tsx. Migration involves incremental adoption — App Router can coexist with Pages Router. Key differences: data fetching (server components vs getServerSideProps), layouts (nested layouts in App Router vs shared in _app).

3
Explain Server Components vs Client Components in Next.js.ExpertP1
Basic Answer

Server Components render on the server, reduce JS bundle, can directly access databases/APIs/filesystem, and never re-render on the client. Client Components ('use client') are interactive — can use hooks, event handlers, browser APIs. Default in App Router is Server Components. Good rule: start with Server Components, add 'use client' only when you need interactivity.

4
Compare SSR, SSG, ISR — when to use each?IntermediateP1
Basic Answer

SSR (Server-Side Rendering) — renders on each request. Use for: personalized content, real-time data. SSG (Static Site Generation) — builds at build time. Use for: marketing pages, docs, blogs. ISR (Incremental Static Regeneration) — static + periodic revalidation. Use for: content that changes but doesn't need to be real-time (ecommerce, blog with updates). Next.js 14+ uses revalidate option or generateStaticParams + revalidate for ISR.

5
How does data fetching work in the App Router?AdvancedP1
Basic Answer

In Server Components, fetch directly: async function Page() { const data = await fetch(url); }. Fetch is extended with caching/revalidation options: cache: 'force-cache' | 'no-store', next: { revalidate: 60 }. generateStaticParams for static params. Parallel data fetching with Promise.all. Sequential data fetching when one depends on another. Previously getServerSideProps/getStaticProps in Pages Router.

6
What are layouts in Next.js App Router? How do they differ from pages?IntermediateP1
Basic Answer

Layouts (layout.tsx) wrap child pages and persist across navigations (no re-render). template.tsx is similar but re-mounts on each navigation. Nested layouts create persistent UI shells — useful for dashboards, settings, multi-step flows. page.tsx is the actual page content that changes between routes.

7
Explain loading UI, error UI, and not-found pages in App Router.IntermediateP1
Basic Answer

loading.tsx — shown while page content loads (triggers on navigation). error.tsx — error boundary for that route segment ('use client' required). not-found.tsx — 404 page for that route. These files create React boundaries that isolate loading and error states to specific route segments, nested in the layout hierarchy.

8
What is middleware in Next.js? Common use cases.AdvancedP1
Basic Answer

Middleware (middleware.ts) runs before every request on the Edge runtime. Use cases: authentication/redirects, redirecting based on country/locale, A/B testing, request headers modification, bot detection, and feature flags. Middleware can match specific paths using config.matcher. It runs on every request, so keep it fast — no heavy computations.

9
How do you handle authentication in Next.js?AdvancedP1
Basic Answer

Options: NextAuth.js/Auth.js (most popular, supports many providers), Clerk (hosted auth), Supabase Auth, Lucia Auth (lightweight), or custom JWT/session auth. Pattern: middleware checks auth, protects routes, redirects to login. Server Components can read session via cookies. API routes validate auth tokens. Next.js 14+ supports middleware-based auth + server-side session validation.

10
What are Route Handlers (API routes) in App Router?IntermediateP1
Basic Answer

Route Handlers (route.ts) replace API routes from Pages Router. They support GET, POST, PUT, PATCH, DELETE, HEAD, OPTIONS. Can read params, searchParams, request body. Support streaming responses, Webhooks, and middleware. They run on the server only — no client bundle.

11
How does Next.js handle images? Explain the Image component.IntermediateP1
Basic Answer

next/image provides automatic optimization: responsive sizes, WebP/AVIF format, lazy loading, placeholder blur, and CLS prevention. Requires width/height or fill. Remote images need remotePatterns in config. Image optimization is handled by the server/Edge at request time.

12
How do you handle metadata and SEO in Next.js?IntermediateP1
Basic Answer

App Router uses export const metadata or generateMetadata() for dynamic metadata. Supports: title, description, open graph, twitter cards, canonical URLs, robots, and more. generateMetadata is async — fetches data for dynamic pages. Server Components allow dynamic meta tags. Previously next/head or NextSeo in Pages Router.

13
What is streaming in Next.js? How does it improve UX?ExpertP1
Basic Answer

Streaming sends HTML progressively from server to client. The page shows loading skeletons while waiting for slow data. loading.tsx provides per-route loading UI. boundaries allow granular streaming. Benefits: faster TTFB (Time to First Byte), better perceived performance, and users see content sooner.

14
How does caching work in Next.js App Router?ExpertP1
Basic Answer

Multiple cache layers: Data Cache (fetch results are cached), Full Route Cache (static HTML), Router Cache (client-side navigation cache). cache: 'force-cache' (SSG), cache: 'no-store' (dynamic), revalidate: X (ISR). revalidatePath() and revalidateTag() for on-demand revalidation. Understanding these layers is crucial for performance optimization.

15
How do you deploy a Next.js application?IntermediateP1
Basic Answer

Vercel (recommended): seamless deployment with automatic ISR, image optimization, and Edge Functions. Self-hosted: Node.js server, Docker image, or static export (next export). Other platforms: AWS (Amplify, ECS), Google Cloud Run, Azure, Netlify. Docker example: multi-stage build with Node.js 18+.

16
What is Partial Prerendering (PPR) in Next.js?ExpertP1
Basic Answer

PPR (experimental in Next.js 14) combines static and dynamic rendering on the same page. Static shell is prerendered at build time, dynamic content is streamed in. Uses Suspense boundaries to separate static/dynamic sections. Gives the performance of SSG with the freshness of SSR. Only available in App Router with specific config.

P2 — Commonly Asked Questions

17
What are parallel routes and intercepting routes?AdvancedP2
Basic Answer

Parallel routes (@modal, @sidebar in folder names) render multiple pages in the same layout simultaneously — great for dashboards. Intercepting routes ((.)folder, (..)folder) intercept navigations and show content in a modal instead of navigating — great for feed-based apps with detail modals.

18
What is generateStaticParams? How does it work with dynamic routes?IntermediateP2
Basic Answer

generateStaticParams returns an array of params to pre-render at build time for dynamic routes ([slug], [id]). Combined with revalidate for ISR. If it returns all possible values, the page can be fully static. If not, Next.js falls back to SSR on first request and caches the result.

19
How do you handle environment variables in Next.js?BeginnerP2
Basic Answer

.env.local (local), .env.production, .env.development. Client variables need NEXT_PUBLIC_ prefix. Server-only variables stay private. In App Router, Server Components can access server env vars directly. Use @next/env for runtime env validation (with Zod).

20
How does font optimization work in Next.js?BeginnerP2
Basic Answer

next/font automatically optimizes Google Fonts and custom fonts. Uses CSS size-adjust to prevent layout shift (CLS). Downloads font files at build time (no external requests). Supports variable fonts for performance. Subset fonts automatically to reduce size.

21
What is the next/script component?IntermediateP2
Basic Answer

Optimizes third-party script loading. Strategies: beforeInteractive (critical), afterInteractive (default — most scripts), lazyOnload (non-critical), worker (experimental — runs in web worker). Controls loading priority and prevents render blocking.

22
How do you implement i18n in Next.js?AdvancedP2
Basic Answer

App Router: built-in i18n with middleware-based routing. Use next-intl or next-i18next for translation management. Middleware detects user locale, redirects to /en/page, /fr/page. Server Components load translations. Dynamic routes with [locale]. Supports static generation for all locales.

23
Explain the component and prefetching.IntermediateP2
Basic Answer

next/link prefetches pages in the background (when link is in viewport). Prefetching is automatic for static routes, lower priority for dynamic routes. prefetch={false} to disable. Client-side transitions are faster than full page loads. Supports scroll (scroll to top), replace (no history push).

24
How do you use useRouter in App Router?IntermediateP2
Basic Answer

useRouter from next/navigation provides: push, replace, back, forward, refresh (re-renders Server Components without losing client state), and prefetch. NOT from next/router (Pages Router). useParams, useSearchParams, usePathname for reading route info.

25
What are generateMetadata and generateViewport?IntermediateP2
Basic Answer

generateMetadata({ params, searchParams }) returns dynamic metadata based on route params — essential for SEO for dynamic pages. generateViewport handles viewport/theme-color/color-scheme. Both are Server-Component functions. Supports generating opengraph images with @vercel/og.

26
What are the different rendering strategies in Next.js?AdvancedP2
Basic Answer

Static (default): HTML generated at build time. Dynamic: rendered per request. Streaming: progressive HTML delivery. ISR: static + periodic revalidation. Edge: runs on Edge Runtime for ultra-low latency. The App Router automatically determines the strategy based on features used (cookies(), headers(), searchParams make routes dynamic).

27
How do you handle redirects in Next.js?IntermediateP2
Basic Answer

next.config.js redirects() — static redirects (301, 308, 307). Middleware redirects — dynamic based on conditions. redirect() in Server Components — server-side redirect. useRouter.push() — client-side navigation. permanentRedirect() for 301. notFound() for 404.

28
What is the next/headers and next/cookies API?AdvancedP2
Basic Answer

headers() — read request headers in Server Components/Route Handlers. cookies() — read/write cookies. Both are dynamic APIs that opt the route into dynamic rendering. Used for: reading authorization tokens, user preferences, A/B test assignments. Available only in Server Components and Route Handlers.

P3 — Occasionally Asked (Valuable Context)

29
How does the use hook work in Next.js Server Components?ExpertP3
Basic Answer

use(promise) (React 19) allows reading a Promise in render — works in both Server and Client Components. In Server Components, it's similar to await. In Client Components, it integrates with Suspense. Enables more flexible data loading patterns.

30
What are server actions in Next.js?ExpertP3
Basic Answer

Server Actions ('use server') are functions that run on the server but can be called from the client. Used for form submissions, data mutations. They simplify the pattern of: submit form → call API → revalidate data. Can be used with React 19's

for progressive enhancement.

31
How does Turbopack differ from Webpack in Next.js?ExpertP3
Basic Answer

Turbopack is the Rust-based successor to Webpack in Next.js. 10-100x faster for development. Written in Rust. Not yet stable for production builds (falls back to Webpack). Enable with --turbo. Supports CSS, CSS Modules, JSX, TypeScript, and dynamic imports.

32
What is the next/og library?AdvancedP3
Basic Answer

@vercel/og (integrated as next/og) generates dynamic Open Graph images at the Edge using Satori (HTML + CSS to image). Use in generateMetadata or Route Handlers. No browser needed — renders images server-side. Essential for dynamic social sharing images per page/product/article.

33
How do you test Next.js applications?IntermediateP3
Basic Answer

Jest + React Testing Library for unit/integration tests. @testing-library/react with Next.js jest config. Playwright for E2E tests — covers full user flows, API routes, and data fetching. Vitest alternative. MSW for API mocking. Focus on integration tests for Server Components and E2E for critical user journeys.

34
Explain incremental adoption — running Pages Router alongside App Router.AdvancedP3
Basic Answer

Next.js supports both routers in the same app. Routes under pages/ use Pages Router, routes under app/ use App Router. Start adapting: migrate shared layouts, then leaf pages. Use next/link to navigate between routers — full page navigation occurs. No shared state between routers.

35
What are Edge Functions vs Serverless Functions in Next.js?ExpertP3
Basic Answer

Edge Functions: run on Edge Runtime (V8 isolates), fast cold start (~50ms), limited Node.js APIs, global distribution. Serverless Functions: full Node.js runtime, more APIs, slower cold start. Middleware always runs on Edge. Route Handlers can be configured to run on Edge or Node.js runtime.
