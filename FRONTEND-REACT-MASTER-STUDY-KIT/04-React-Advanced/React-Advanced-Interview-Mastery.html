P1 — Frequently Asked Questions

Must-know for Senior React roles.

1
Explain React.memo — when should you actually use it?AdvancedP1
Basic Answer

React.memo(Component, areEqual?) prevents re-renders when props haven't changed (shallow comparison). Use when: component re-renders frequently with same props, component renders expensive content (charts, maps), or it has many children. DON'T use on every component — the comparison itself has cost. Profile first with React DevTools to identify unnecessary re-renders.

2
Explain useMemo vs useCallback — when to use each and when NOT to.AdvancedP1
Basic Answer

useMemo memoizes a computed value — use for expensive calculations (filtering 10k items, complex derivations). useCallback memoizes a function reference — use when passing to memoized children. When NOT to use: trivial calculations (the comparison overhead exceeds computation cost), always-changing deps (memoization is wasted), in hot paths (additional function call overhead). React docs: "Do not rely on it for correctness — it's a performance optimization."

3
What are custom hook patterns? Design patterns for reusable hooks.AdvancedP1
Basic Answer

Patterns: (1) State + Effect hooks — useLocalStorage, useDebounce, useMediaQuery. (2) Event listener hooks — useEventListener, useClickOutside. (3) Data fetching hooks — useFetch, useApi. (4) State machine hooks — useToggle, useCounter. (5) Compound hooks — multiple hooks that work together. Best practices: return arrays for primitive hooks (like useState), objects for complex hooks, accept options objects for extensibility.

function useLocalStorage(key: string, initial: T) { const [value, setValue] = useState(() => { const stored = localStorage.getItem(key); return stored ? JSON.parse(stored) : initial; }); useEffect(() => { localStorage.setItem(key, JSON.stringify(value)); }, [key, value]); return [value, setValue] as const; }
4
What is the stale closure problem in React hooks? How to fix it?ExpertP1
Basic Answer

Stale closures occur when async code captures outdated state/props. Common scenarios: setTimeout in useEffect without proper deps, setInterval capturing old state, event listeners attached in useEffect. Fixes: (1) Include all deps in dependency array. (2) Use functional updater setCount(prev => prev + 1). (3) Use useRef to store the latest value. (4) Use useCallback with stable deps.

5
What is the difference between useEffect and useLayoutEffect?AdvancedP1
Basic Answer

useEffect runs after the browser has painted (async). useLayoutEffect runs synchronously BEFORE paint — the browser waits for it to finish. Use useLayoutEffect when you need to read/measure DOM layout and make visual changes before the user sees them (animations, tooltip positioning, scroll adjustments). Prefer useEffect by default for most cases.

6
Explain Error Boundaries — implement one with componentDidCatch.AdvancedP1
Basic Answer

Error Boundaries catch rendering errors in child components. Must be a class component (React 18 and earlier). Use getDerivedStateFromError to update state (render fallback) and componentDidCatch for logging. React 19 introduces error boundaries for functional components. Place error boundaries strategically around major sections of your app (each route, widget, independent section).

7
What is Suspense? How does it work with lazy and data fetching?AdvancedP1
Basic Answer

Suspense lets you show a fallback while content is loading. Currently used with React.lazy for code splitting. In React 18+, Suspense works with data fetching libraries (Relay, SWR, React Query via throw promise pattern). React 19 expands Suspense for Server Components and streaming SSR. Key: Suspense boundaries let you compose loading states granularly.

8
Explain render props vs HOCs vs hooks — compare and contrast.AdvancedP1
Basic Answer

Render props: }/> — flexible but creates wrapper hell. HOCs: withFeature(Component) — wrapper hell, naming collisions, static property issues. Hooks: const data = useFeature() — no wrapper hell, composable, no naming collisions, TypeScript-friendly. Hooks are the modern, recommended approach. Some libraries still use HOCs for context-specific needs.

9
How do you optimize a React app with large lists? (Windowing/Virtualization)AdvancedP1
Basic Answer

Virtualization renders only visible items + a small buffer. Popular libraries: react-window (lightweight, fixed-size), react-virtuoso (auto-size, features), and tanstack-virtual (headless). Key considerations: item height estimation, scroll restoration, dynamic heights, list padding for scrollbar. For 10k+ items, virtualization is essential for performance.

10
What are concurrent features in React 18? (Transitions, Suspense, Automatic Batching)ExpertP1
Basic Answer

Concurrent React can interrupt, pause, or abort rendering. Features: useTransition — marks low-priority updates (search results). useDeferredValue — defers a value's update. startTransition — for non-hook contexts. Automatic Batching — batches all state updates (even in promises/setTimeout). Suspense for data fetching. These improve perceived performance by keeping UI responsive during expensive updates.

What is the ref callback pattern? When would you need it instead of useRef?AdvancedP1
Basic Answer

Ref callbacks:

Ref callbacks are called with null on unmount and with the element on mount. They're more flexible than useRef for dynamic scenarios.

12
What are controlled vs uncontrolled components for forms?IntermediateP1
Basic Answer

Controlled: React state manages value — value={state} onChange={handler}. Predictable, enables validation, conditional UI, and instant feedback. Uncontrolled: DOM manages value — use useRef to read. Simpler, better performance for large forms, works with file inputs. React Hook Form uses uncontrolled approach with refs for performance.

13
How does React handle re-renders? What triggers them and how to prevent unnecessary ones?AdvancedP1
Basic Answer

Triggers: state changes, parent re-renders, context changes. Prevention: React.memo for pure components, lift state down (keep state as local as possible), use useMemo/useCallback for stable references, avoid creating new objects/arrays in render (inline styles, object props), use keys to preserve identity, split components to localize re-renders.

14
What is the useId hook? When would you use it?AdvancedP1
Basic Answer

useId() generates unique, stable IDs for accessibility attributes. Prevents hydration mismatches between server and client. Use for: htmlFor/id pairs in forms, aria-describedby, aria-labelledby. Much better than Math.random() or incrementing counters (which cause hydration issues). Available in React 18+.

15
Explain the concept of "rendering" vs "committing" in React.ExpertP1
Basic Answer

Render phase: React calls your component function, creates Virtual DOM tree, runs reconciliation (diffing). Can be paused, interrupted, or discarded (concurrent features). Commit phase: React applies DOM mutations, runs useLayoutEffect (sync), then useEffect (async). Rendering should be pure — no side effects. Side effects belong in useEffect (after commit).

16
How do you handle side effects in React? Compare approaches.IntermediateP1
Basic Answer

useEffect — primary approach for syncing with external systems. useLayoutEffect — for DOM measurements/layout changes before paint. Custom hooks — encapsulate side effect logic. Event handlers — side effects triggered by user actions (not component lifecycle). Data fetching libraries (React Query, SWR) — abstract side effects with caching, dedup, refetching. Next.js Server Components — run side effects on the server.

17
What is the useDebugValue hook used for?AdvancedP1
Basic Answer

useDebugValue(value) labels custom hooks in React DevTools. Used inside custom hooks to display debug information. Accepts a formatter function for expensive formatting (only called when DevTools is open). Improves developer experience when building shared hook libraries.

18
How do you test React hooks? What approaches work?AdvancedP1
Basic Answer

renderHook from @testing-library/react-hooks (or built-in in RTL v14+). Test: initial state, state updates, side effects (use mock functions). act() wraps state updates. Alternatives: test hooks indirectly through components that use them (integration testing). Prefer integration tests for custom hooks — test the behavior, not the hook internals.

P2 — Commonly Asked Questions

19
What is the difference between useEffect cleanup and useEffect with empty deps?IntermediateP2
Basic Answer

Cleanup runs on unmount AND before re-running the effect (when deps change). Empty deps [] runs once on mount, cleanup runs once on unmount. Proper cleanup prevents memory leaks (event listeners, timers, subscriptions, AbortController).

20
How do you implement componentDidUpdate logic with hooks?IntermediateP2
Basic Answer

useEffect with deps runs when those deps change. To run on update (not initial mount), use a ref to track whether it's the first render: const isFirst = useRef(true); if (isFirst.current) { isFirst.current = false; return; }. Or use useEffect(() => {}, deps) which runs after every render where deps changed (including mount).

21
How do you abort fetch requests in React when a component unmounts?IntermediateP2
Basic Answer

Use AbortController in useEffect — call abort() in cleanup. React Query/SWR handle this automatically. Without cancellation, updates on unmounted components cause memory leaks and "setState on unmounted component" warnings (React 18 shows no warning, but the fetch continues unnecessarily).

useEffect(() => { const controller = new AbortController(); fetch(url, { signal: controller.signal }) .then(r => r.json()) .then(setData) .catch(err => { if (err.name !== 'AbortError') setError(err); }); return () => controller.abort(); }, [url]);
22
What is the useEvent hook (React 19)?ExpertP2
Basic Answer

useEvent(callback) (React 19 canary) creates a stable function reference that always has the latest props/state. Unlike useCallback, it doesn't need dependencies — the callback "sees" current values without re-creation. Useful for event handlers and callbacks in useEffect that should always use latest values. Still experimental.

23
What is the useOptimistic hook?ExpertP2
Basic Answer

useOptimistic (React 19) enables optimistic updates — you can immediately show the expected result while the server processes the request. If the server confirms, the optimistic state persists. If the server rejects, it reverts to the actual state. Essential for chat apps, likes, form submissions — any UX that benefits from instant feedback.

24
How do you handle focus management in React?IntermediateP2
Basic Answer

Use autoFocus prop for initial focus. Use useRef + ref.current.focus() for programmatic focus. Use forwardRef to pass refs to custom components. For complex focus management (modals, menus), use the focus-trap-react library or implement :focus-visible CSS.

25
What is the useSyncExternalStore hook?ExpertP2
Basic Answer

useSyncExternalStore(subscribe, getSnapshot, getServerSnapshot?) allows reading from external stores (Redux, Zustand) with concurrent safety. Ensures consistency during concurrent rendering — prevents "tearing" where different parts of the UI see different store values. Used internally by Redux and Zustand. Not needed for most application code.

Explain the concept of "lifting content up" for performance.AdvancedP2
Basic Answer

"Lifting content up" means passing heavy JSX as children or props so it doesn't re-render when the parent re-renders. The parent's re-render doesn't affect the children prop — the child only re-renders if its own state/props change. This is a powerful pattern for isolating re-renders without React.memo.

How do you handle animations in React?IntermediateP2
Basic Answer

CSS transitions/animations — simplest for basic effects. Framer Motion — most popular React animation library (layout animations, gestures, variants). React Spring — physics-based animations. GSAP — powerful but less React-idiomatic. CSS-in-JS with keyframes. For mount/unmount animations, use AnimatePresence (Framer Motion) or CSSTransition (react-transition-group).

What is the dangerouslySetInnerHTML prop? When would you use it?IntermediateP2
Basic Answer

Replaces innerHTML. Only use with sanitized content (use DOMPurify). Alternatives: parse HTML to React components (html-react-parser), use Markdown libraries for rich text, render structured data directly. Avoid if at all possible — it bypasses React's XSS protection.

30
Explain server components in React.ExpertP2
Basic Answer

Server Components run ONCE on the server, never on the client. They can access databases, file systems, and backend APIs directly. They reduce client bundle size (heavy deps stay on server). They cannot use state, effects, or event handlers. Client Components (with 'use client') are for interactivity. Next.js App Router uses this model. Benefits: smaller bundles, direct backend access, automatic code splitting.

action attribute in React 19?ExpertP2
Basic Answer

React 19 introduces action prop on that automatically manages onSubmit and pending state. Combined with useFormStatus and useOptimistic, it provides a full form handling system. useFormStatus gives the pending state of the parent form. This simplifies form UX significantly.

32
How does React DevTools profiling work? What metrics matter?IntermediateP2
Basic Answer

React DevTools Profiler records component renders. Key metrics: render duration, why a component rendered (state, props, context, parent render), commit phases. Look for: components re-rendering unnecessarily, slow renders, large component subtrees re-rendering on small state changes. Use the "flamegraph" view and "ranked" view to identify bottlenecks.

P3 — Occasionally Asked (Valuable Context)

33
What is React's createPortal and how does event bubbling work through portals?AdvancedP3
Basic Answer

createPortal renders children to a different DOM node. Event bubbling follows the React tree, not the DOM tree. So a click on a portal child bubbles up to the React parent (not the DOM parent that contains the portal target). This is crucial for context providers and event handlers to work correctly with portals.

34
How would you build a compound component pattern?ExpertP3
Basic Answer

Compound components share implicit state: . Implementation: Context for shared state, React.Children.map for passing props to children, cloneElement for injecting props (use sparingly). Libraries: @reach-ui, headless-ui, radix-ui provide accessible compound components.

35
What is the useInsertionEffect hook?ExpertP3
Basic Answer

useInsertionEffect runs before DOM mutations are committed. Designed for CSS-in-JS libraries to inject styles before React paints. Library authors use it — application code shouldn't. It has no access to refs or DOM state.

36
How do you migrate a class component to hooks?AdvancedP3
Basic Answer

Step by step: (1) Convert to function. (2) Replace this.state with useState/useReducer. (3) Replace lifecycle methods with useEffect. (4) Replace this.method = this.method.bind(this) with arrow functions. (5) Replace refs with useRef. (6) Replace context with useContext. Test each step. Be careful with componentDidUpdate — you may need multiple useEffects.

37
How do you handle WebSocket connections in React?AdvancedP3
Basic Answer

Create WebSocket in useEffect, clean up in return. Handle reconnection, heartbeat, and message buffering. Consider using libraries like socket.io-client or Pusher for production. Custom hook useWebSocket can encapsulate connection lifecycle. Remember to handle: connection states (connecting, open, closing, closed), error recovery, and race conditions with component updates.

38
What is React.startTransition? When would you use it outside a component?ExpertP3
Basic Answer

startTransition(callback) marks state updates as non-urgent. Used outside hooks — in callback props, store actions, or libraries. React will prioritize urgent updates (input changes) over transitions (filtering a list). Combines with useTransition for isPending state.

39
How does React handle context updates? What's the performance impact?ExpertP3
Basic Answer

Context value changes cause ALL consumers to re-render, even if they only read a part of the value. Performance solutions: (1) Split context into smaller pieces (separate concerns). (2) Use useMemo for context value to prevent unnecessary updates. (3) Use libraries like Zustand or Jotai that solve this with selector-based subscriptions. (4) Use useContextSelector (third-party) for fine-grained subscriptions.

40
How do you test components that use useEffect with async operations?AdvancedP3
Basic Answer

Mock fetch/API calls. Use waitFor, findBy* queries for async assertions. Use act to wrap state updates. Use jest.useFakeTimers for timer-based effects. Wait for loading states to resolve before asserting. MSW (Mock Service Worker) is the recommended approach for mocking API calls during testing.
