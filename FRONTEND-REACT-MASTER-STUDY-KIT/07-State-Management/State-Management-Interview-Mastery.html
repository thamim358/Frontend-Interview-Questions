P1 — Frequently Asked Questions

1
What are the different types of state in a React application?IntermediateP1
Basic Answer

Local state (useState/useReducer), Global state (Context, Zustand, Redux), Server state (React Query, SWR), URL state (search params, pathname), Form state (React Hook Form), and URL/persisted state. Each type has the right tool. Most apps need 2-3 types. Server state is the most commonly misunderstood — developers often put it in Redux when it belongs in a caching layer.

2
Compare Redux Toolkit, Zustand, and Context API. When to use each?AdvancedP1
Basic Answer

Context API: built-in, good for low-frequency updates (theme, auth, locale). Performance issue: all consumers re-render on any value change. Zustand: minimal boilerplate, selector-based subscriptions (performant), no provider needed, TS-friendly, middleware support. Redux Toolkit: structured, DevTools, middleware ecosystem, RTK Query for data fetching. Choose: small app → Context + hooks. Medium → Zustand. Large/team → Redux Toolkit. Server state always → React Query.

3
What is React Query (TanStack Query)? How does it simplify data fetching?AdvancedP1
Basic Answer

React Query manages server state — caching, background refetching, deduplication, pagination, optimistic updates, and stale-while-revalidate. Key concepts: useQuery (read), useMutation (write), queryClient (cache management), stale time vs cache time. Eliminates the need to put server data in Redux/Zustand. Provides loading/error states automatically. DevTools for debugging queries.

4
How does Redux Toolkit differ from vanilla Redux?AdvancedP1
Basic Answer

RTK reduces boilerplate dramatically. createSlice generates actions and reducers automatically (uses Immer internally for immutable updates). configureStore sets up middleware (thunk by default) and DevTools. createAsyncThunk for async actions. RTK Query for API caching. Vanilla Redux: manual action creators, switch-case reducers, immutable spread operators. RTK is the officially recommended approach.

5
What is Zustand? How does it compare to Redux for global state?AdvancedP1
Basic Answer

Zustand is a tiny (~1KB) state management library with no boilerplate. Creates a store as a hook: const useStore = create(set => ({ count: 0, inc: () => set(s => ({ count: s.count + 1 })) })). Selectors prevent unnecessary re-renders. Middleware support (persist, immer, devtools). No Provider needed. TS-friendly. Redux is more structured (great for teams), Zustand is simpler (great for speed).

6
Explain the Context API — when is it NOT enough?IntermediateP1
Basic Answer

Context is NOT enough when: (1) State changes frequently (all consumers re-render). (2) You need fine-grained subscriptions (can't select context slices). (3) Complex state logic with multiple reducers. (4) DevTools/time-travel debugging. (5) Multiple stores. Solutions: split contexts, use useMemo for context value, or use Zustand/Jotai for better performance. Context is for dependency injection (theme, auth), not state management at scale.

7
What is server state vs client state? Why does the distinction matter?IntermediateP1
Basic Answer

Server state: data from the backend (user profiles, posts, products). Async, needs caching, stale-while-revalidate, background refetching. Client state: UI state (modals open, form inputs, selected filters). Sync, instant, local. Don't put server state in Redux/Zustand — use React Query or SWR. They handle caching, dedup, retries, and refetching that client state managers don't.

8
What are selectors and why are they important?IntermediateP1
Basic Answer

Selectors extract/compute derived state from the store. Prevent unnecessary re-renders by subscribing to specific slices. createSelector (Reselect/RTK) enables memoized selectors — recompute only when inputs change. In Zustand: const name = useStore(s => s.user.name). Selectors are crucial for performance in large applications.

9
What is Immer? How does it simplify immutable state updates?AdvancedP1
Basic Answer

Immer (produce) lets you write mutable-looking code that produces immutable state. Uses Proxies to track mutations and produce a new immutable state. Built into Redux Toolkit. Example: state.users[0].name = 'New' — Immer handles deep cloning. Without Immer: { ...state, users: state.users.map((u, i) => i === 0 ? { ...u, name: 'New' } : u) }.

10
What are middleware in Redux? Common examples?AdvancedP1
Basic Answer

Middleware intercepts dispatched actions before they reach the reducer. Common middleware: Redux Thunk (async actions — built into RTK), Redux Logger (debugging), redux-persist (persist state), redux-saga (side effects with generators — legacy). Zustand middleware: persist, immer, devtools. Middleware enables extensibility without modifying core logic.

11
What is Jotai? How does atomic state differ from Redux/Zustand?AdvancedP1
Basic Answer

Jotai uses atomic state — minimal, independent state units that can be composed. const countAtom = atom(0); const doubledAtom = atom(get => get(countAtom) * 2). No store object, no reducers. Derived atoms auto-recompute. Fine-grained subscriptions — only components using specific atoms re-render. Good for complex state dependencies. Similar to Recoil but simpler API, smaller bundle.

12
Explain optimistic updates. How do you implement them with React Query?ExpertP1
Basic Answer

Optimistic updates immediately show the expected result before server confirms. React Query: onMutate — set query data optimistically, onError — rollback to previous state, onSettled — refetch to ensure consistency. Use queryClient.setQueryData for immediate update and queryClient.cancelQueries to prevent race conditions.

13
How do you persist state across sessions?IntermediateP1
Basic Answer

localStorage: custom hook useLocalStorage, Zustand persist middleware, Redux redux-persist. IndexedDB for larger data. URL/search params for shareable state. Avoid persisting server state (it will be refetched). Only persist UI state that should survive page reloads.

14
How do you handle loading and error states with global state?IntermediateP1
Basic Answer

Discriminated unions: type AsyncState = { status: 'idle' } | { status: 'loading' } | { status: 'success'; data: T } | { status: 'error'; error: string }. React Query handles this automatically. For Zustand/Redux: manage status alongside data. Always handle all states — loading skeletons, error boundaries, empty states.

15
What is the difference between useMemo and selector functions?AdvancedP1
Basic Answer

Both memoize values. useMemo is local to a component — recalculates when component re-renders and deps change. Selectors memoize at the store level — recomputed only when the relevant slice of state changes. Selectors are shared across components. Use selectors for derived global state, useMemo for expensive local computations.

P2 — Commonly Asked Questions

16
What is the Flux pattern? How does Redux implement it?IntermediateP2
Basic Answer

Flux: unidirectional data flow — Action → Dispatcher → Store → View. Redux simplifies Flux: single store, pure reducers, dispatch actions. Action → Reducer → Store → View. Redux enforces immutability and predictability. The pattern makes state changes traceable and debuggable.

17
What is createAsyncThunk in Redux Toolkit?AdvancedP2
Basic Answer

createAsyncThunk generates action types (pending, fulfilled, rejected) for async logic. Handles loading/error states automatically. Integrates with extraReducers in slices. Example: export const fetchUsers = createAsyncThunk('users/fetch', async () => { const res = await fetch('/api/users'); return res.json(); }).

18
What is the difference between useQuery and useMutation?IntermediateP2
Basic Answer

useQuery for reading data — auto-fetches, caches, refetches. useMutation for creating/updating/deleting — manual trigger, onSuccess/onError/onSettled callbacks, optimistic updates. Mutations invalidate queries to trigger refetch. useMutation doesn't retry by default (unlike queries).

19
What is Zustand's subscribe API used for?AdvancedP2
Basic Answer

useStore.subscribe(selector, callback) allows subscribing to store changes outside React (in event handlers, callbacks, or non-React code). Useful for: analytics tracking, syncing with external systems, logging, and inter-window communication. Unsubscribing returns a cleanup function.

20
How do you handle form state in React? Compare libraries.IntermediateP2
Basic Answer

React Hook Form (most popular) — uncontrolled refs, minimal re-renders, Zod/Yup validation. Formik — more boilerplate, controlled. Final Form — subscription-based. Manual useState — for simple forms only. React Hook Form + Zod is the recommended stack for 2024.

21
What is URL state management? When should you use it?AdvancedP2
Basic Answer

URL state stores state in the URL (search params, path). Makes state shareable, bookmarkable, and persists across refreshes. Use for: filters, search queries, pagination, sorting, selected tabs. Next.js: useSearchParams + useRouter. React Router: useSearchParams.

22
What is stale-while-revalidate pattern?IntermediateP2
Basic Answer

Show stale (cached) data immediately, then refetch in the background. Updates the UI when fresh data arrives. React Query's default strategy: staleTime (how long data is fresh) + cacheTime (how long unused data stays in cache). Balances responsiveness (instant UI) with freshness (up-to-date data).

23
How do you test state management logic?AdvancedP2
Basic Answer

Reducers/slices: pure functions — easy to unit test. Actions/thunks: test with mock store. Zustand stores: test outside React — call store actions, assert state. React Query: test components with QueryClientProvider. Integration tests (React Testing Library) catch real issues. Don't test state management internals directly — test behavior.

24
What is the Actor Model in state management (XState)?ExpertP2
Basic Answer

XState uses state machines/statecharts — explicit states, transitions, guards, and actions. Benefits: all possible states are explicit, impossible states are impossible, visual debugging. Overkill for most apps but excellent for complex flows (multi-step forms, wizards, checkout, game logic). Better than boolean flags for modeling complex state.

25
What is re-reselect and when would you need it?ExpertP2
Basic Answer

re-reselect creates selectors that are memoized by key. Standard Reselect selectors share one cache — if called with different params across components, they recompute. re-reselect keeps separate caches per key. Useful for: list selectors by ID, reusable selectors with parameters in large apps.

26
Compare MobX vs Redux Toolkit.AdvancedP2
Basic Answer

MobX: mutable, reactive, observable-based, automatic tracking, less boilerplate. Redux Toolkit: immutable, functional, explicit actions, structured. MobX is more flexible (can lead to less predictable code), Redux is more structured (better for teams). MobX uses proxies for reactivity. Both have DevTools. Redux is more popular in 2024.

27
What is Recoil? Why isn't it as popular as Jotai/Zustand?AdvancedP2
Basic Answer

Recoil (Meta) pioneered atomic state in React. atom + selector pattern. Jotai is simpler (no atom families, no RecoilRoot needed). Zustand is simpler (single store with selectors). Recoil is experimental, has stability concerns, and received less maintenance. Jotai became the de facto atomic state library.

28
How does Redux DevTools help with debugging?IntermediateP2
Basic Answer

Features: time-travel debugging, action log, state diffing, dispatching actions manually, dependency graph. Helps debug: unexpected state changes, incorrect action flows, and performance issues (number of re-renders). RTK includes DevTools middleware automatically.

29
What is queryClient.invalidateQueries?IntermediateP2
Basic Answer

Marks cached queries as stale, triggers refetch. Used after mutations to update the UI. queryClient.invalidateQueries({ queryKey: ['users'] }). Can target specific queries with filters. refetchType: 'active' only refetches active queries. Essential for keeping server state in sync after writes.

P3 — Occasionally Asked (Valuable Context)

30
What is normalizr and when would you use it?ExpertP3
Basic Answer

Normalizr normalizes nested API responses into flat entities. Eliminates duplicate data and enables O(1) lookups. normalize(data, [user]) → { entities: { users: { 1: {...} } }, result: [1] }. Useful for: complex relational data, Redux stores, and caching layers. Redux Toolkit's createEntityAdapter provides similar functionality.

31
How does CSS-in-JS affect server-side rendering state?AdvancedP3
Basic Answer

CSS-in-JS libraries inject styles at runtime. For SSR, they need server-side collection to inject styles into HTML. ServerStyleSheet (Styled Components) collects styles during render. extractCritical (Emotion). If not handled, first paint has no styles (FOUC). Next.js App Router handles this better with CSS Modules and Tailwind.

32
What is valtio and how does it differ from other state managers?AdvancedP3
Basic Answer

Valtio (by the Jotai team) uses proxy-based mutable state. const state = proxy({ count: 0 }) — mutate directly: state.count++. Components auto-track accessed properties. Snapshot API for immutable reads. Different from Zustand (immutable API) and Jotai (atomic). Best for: apps that prefer mutable mental model.

33
How do you implement undo/redo with Redux Toolkit?ExpertP3
Basic Answer

Pattern: wrap reducer with undo history. Store past states, future states. UNDO action → move current to future, pop from past. REDO → move current to past, pop from future. redux-undo library or custom middleware. Consider performance — don't store entire state for every action. Debounce frequent updates.

34
What is sync-tabs and how do you sync state across tabs?ExpertP3
Basic Answer

BroadcastChannel API for cross-tab messaging. storage events for localStorage changes. Zustand persist middleware with cross-tab sync. Redux redux-state-sync. Pattern: one tab dispatches change, other tabs receive notification and update. Use for: auth state, theme, preferences.

35
How do you handle real-time state with WebSockets?ExpertP3
Basic Answer

WebSocket → update React Query cache directly (queryClient.setQueryData). Zustand: subscribe to WebSocket, update store. Redux: dispatch actions from WebSocket messages. Libraries: Socket.io-client, Pusher, Supabase Realtime. Key: handle reconnection, message ordering, and stale connection detection. React Query subscriptions experimental.
